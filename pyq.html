What is the significance of standard deviation in Machine Learning?
Unit 4 — Statistical Learning StatisticsFeature Scaling
2 Marks +
✦ Model Answer
Standard Deviation in Machine Learning

Standard Deviation (σ) measures the spread or dispersion of data points around the mean. In ML, it plays several critical roles:

  • Feature Scaling: Used in Z-score normalisation — z = (x − μ)/σ — ensures all features contribute equally. Without it, features with large ranges dominate distance-based algorithms (KNN, SVM).
  • Outlier Detection: Data points beyond ±3σ from the mean are typically outliers. Removing them improves model accuracy.
  • Model Stability: High σ in a feature = high variability = the feature is informative. Very low σ = near-constant feature = can be dropped (low variance filter).
  • Gaussian Assumption: Many ML models (Naive Bayes, GMM) assume features follow a Gaussian distribution — σ is a direct parameter of that distribution.
Formulaσ = √[ Σ(xᵢ − μ)² / N ] where μ = mean, N = number of data points For 2-mark answer: Write definition + formula + one application (e.g., feature scaling).

✦ Model Answer

A Confusion Matrix is an N×N table that summarises classifier performance by comparing actual vs. predicted class labels. For binary classification (N=2):

Confusion Matrix Layout (2×2) Predicted + Predicted − Actual + | TP (True Pos) | FN (False Neg) | Actual − | FP (False Pos) | TN (True Neg) | Accuracy = (TP + TN) / Total Precision = TP / (TP + FP) ← How many predicted + are actually +? Recall = TP / (TP + FN) ← How many actual + did we catch? F1-Score = 2 × (P × R) / (P + R) ← Harmonic mean of P and R FP = Type I Error (False Alarm) · FN = Type II Error (Missed Detection)

What is the working principle of Naive Bayes Classifier?
Unit 2 — Supervised LearningNaive BayesProbabilistic
2 Marks +
✦ Model Answer

Naive Bayes is a probabilistic classifier based on Bayes' Theorem with a "naive" assumption of conditional independence among features given the class label.

Bayes' Theorem — Core FormulaP(C|X) = [ P(X|C) × P(C) ] / P(X) Where: P(C|X) = Posterior — probability of class C given features X ← We want this P(X|C) = Likelihood — probability of features given class C P(C) = Prior — overall probability of class C in training data P(X) = Evidence — constant, same for all classes (ignored in comparison) Naive Assumption: P(X|C) = P(x₁|C) × P(x₂|C) × ... × P(xₙ|C) → Features are treated as independent given the class (hence "Naive") Predicted class = argmax_C [ P(C) × Π P(xᵢ|C) ]

Why "Naive"? Features in real data ARE correlated (e.g., age and income), but Naive Bayes ignores this — treating each feature independently. Despite this oversimplification, it works surprisingly well for text classification.

⚡
Connection to Unit 5

The Bayes' theorem here is the same probabilistic foundation used in Bayesian RL (Unit 5–6). Understanding P(C|X) now will help you understand belief states in MDPs later.

Q9
What does Mean Squared Error (MSE) tell you?
Unit 2 — Supervised LearningUnit 4 — Model EvaluationRegression Metric
2 Marks +
✦ Model Answer

Mean Squared Error (MSE) is the average of the squared differences between predicted and actual values. It is the primary loss function for regression problems.

MSE FormulaMSE = (1/n) × Σᵢ (yᵢ − ŷᵢ)² Where: yᵢ = actual value · ŷᵢ = predicted value · n = number of data points Related Metrics: RMSE (Root MSE) = √MSE ← Same units as output, easier to interpret MAE (Mean Abs Error) = (1/n) × Σ|yᵢ − ŷᵢ| ← Less sensitive to outliers MSE penalises large errors more (squared) → more sensitive to outliers than MAE

What MSE tells you: MSE = 0 means perfect prediction. Higher MSE means greater average prediction error. Since errors are squared, a single large error (outlier) contributes disproportionately — this is both a strength (forces model to reduce big errors) and a weakness (outlier sensitivity).

Deep Learning — Definition

Deep Learning is a subset of Machine Learning that uses Artificial Neural Networks (ANNs) with multiple hidden layers — hence "deep" — to learn hierarchical representations of data automatically from raw inputs.

📊 DEEP LEARNING ARCHITECTURE (Draw This)

Each layer = Neurons + Weights + Activation function
Shallow ANN: 1 hidden layer · Deep ANN: 2+ hidden layers
How Deep Learning Works — Key Concepts
  • Feature Hierarchy: Each layer learns increasingly abstract features. In image recognition: Layer 1 → edges, Layer 2 → shapes, Layer 3 → object parts, Output → full object.
  • Forward Pass: Input flows through layers. At each neuron: z = Σ(wᵢxᵢ) + b, then activation: a = f(z)
  • Backpropagation: Error flows backward, updating weights using gradient descent to minimise loss.
  • Activation Functions: ReLU (most common), Sigmoid, Tanh, Softmax (output for classification).
Key FormulaOutput of neuron: aᵢ = f(Σⱼ wᵢⱼ · aⱼ + bᵢ) where f = activation function Weight update (Gradient Descent): wᵢⱼ ← wᵢⱼ − α · (∂L/∂wᵢⱼ) α = learning rate · L = loss function ReLU: f(x) = max(0, x) · Sigmoid: f(x) = 1/(1+e⁻ˣ)

Shallow ANN (1 hidden layer)Deep ANN (2+ hidden layers)
Requires manual feature engineeringLearns features automatically
Struggles with complex patterns (images, speech)Excels at complex patterns
Fewer parameters, faster trainingMore parameters, needs more data and GPU

Reinforcement Learning (RL) — Definition

Reinforcement Learning is a type of ML where an agent learns to make decisions by interacting with an environment. The agent receives rewards for good actions and penalties for bad ones, and the goal is to learn a policy (action selection strategy) that maximises cumulative reward over time.

📊 RL LOOP (Draw This)
Agent → Action (aₜ) → Environment → Reward (rₜ) + Next State (sₜ₊₁) → Agent
The agent observes state sₜ, takes action aₜ, receives reward rₜ, and sees new state sₜ₊₁
Key RL ComponentsState (s) : Current situation of the agent Action (a) : What the agent does Reward (r) : Signal from environment (positive/negative) Policy (π) : π(s) → a — maps states to actions Value V(s) : Expected cumulative reward from state s following policy π Return (G) : G = r₁ + γr₂ + γ²r₃ + ... where γ ∈ [0,1] = discount factor
Model-Based RL

In Model-Based RL, the agent builds or is given an explicit model of the environment — a transition function T(s, a, s') and reward function R(s, a) — and uses this model to plan actions without necessarily interacting with the real environment repeatedly.

AspectModel-Free RLModel-Based RL
Environment ModelNot used — learns from direct experienceBuilds/uses explicit model T(s,a,s')
PlanningNo planning (reactive)Plans ahead using the model (Dyna, World Models)
Sample EfficiencyNeeds many real interactionsMore efficient — simulates experience from model
ExamplesQ-Learning, SARSA, Policy GradientDyna-Q, AlphaZero (partial), Model Predictive Control
RiskSlow convergenceWrong model → wrong behaviour (model bias)

Model-Based RL Algorithm (Dyna-Q concept): (1) Interact with environment → update Q-values. (2) Update environment model. (3) Simulate experiences from model → update Q-values further. This hybrid approach dramatically speeds up learning.

Markov Decision Process (MDP)

An MDP is a mathematical framework for sequential decision-making. It is defined by the tuple (S, A, T, R, γ):

MDP Formal DefinitionS = Set of states A = Set of actions T(s, a, s') = P(s'|s,a) = Transition probability (Markov Property) R(s, a) = Immediate reward for taking action a in state s γ ∈ [0,1] = Discount factor (how much future rewards are valued) Markov Property: P(sₜ₊₁|sₜ,aₜ) — future depends only on CURRENT state, not history Bellman Equation: V*(s) = max_a [R(s,a) + γ Σs' T(s,a,s') V*(s')]
Policy Iteration

Policy Iteration alternates between two steps until the policy stops changing:

  1. Policy Evaluation: Given policy π, compute V^π(s) for all states by solving the Bellman equation iteratively until convergence.
  2. Policy Improvement: Update policy — for each state, choose the action that maximises expected value: π'(s) = argmax_a [R(s,a) + γ Σs' T(s,a,s') V^π(s')]
  3. Repeat until π' = π (policy is stable = optimal)
Value Iteration

Value Iteration directly computes the optimal value function V*(s) without maintaining an explicit policy until convergence:

Value Iteration Update RuleVₖ₊₁(s) = max_a [ R(s,a) + γ Σs' T(s,a,s') · Vₖ(s') ] Repeat for all states s until |Vₖ₊₁(s) − Vₖ(s)| < ε (convergence threshold) Then extract policy: π*(s) = argmax_a [ R(s,a) + γ Σs' T(s,a,s') · V*(s') ]
AspectPolicy IterationValue Iteration
Maintains explicit policy?Yes, updates π at each stepNo, extracts π only at end
Convergence speedFewer iterations (fast policy convergence)More iterations but each is simple
Computation per stepHeavy (full policy evaluation)Light (single Bellman update)

Unit 4 — Statistical LearningFeature SelectionFilter/Wrapper/Embedded
4 Marks +

Feature Selection — Definition

Feature Selection is the process of selecting a subset of the most relevant input features from a dataset to use in model building — discarding irrelevant, redundant, or noisy features. Goal: reduce dimensionality, improve model accuracy, reduce training time.

Why needed? "Curse of dimensionality" — too many features → overfitting, slow training, hard to visualise. Feature selection counteracts this.

Three Methods
1. Filter Methods (Independent of ML model)

Evaluate features using statistical scores — no ML model involved. Fast but ignores feature interaction.

  • Correlation coefficient — remove features highly correlated with each other
  • Information Gain / Mutual Information — how much does each feature reduce uncertainty about the class?
  • Chi-square test — measures independence between feature and class label
  • Variance threshold — remove features with near-zero variance (constant features)
2. Wrapper Methods (Model-dependent)

Use the ML model itself to evaluate subsets. Expensive but considers feature interactions.

  • Forward Selection: Start with no features → add one at a time → keep if improves accuracy
  • Backward Elimination: Start with all features → remove one at a time → remove if accuracy doesn't drop
  • Recursive Feature Elimination (RFE): Train model → rank features by importance → eliminate weakest → repeat
3. Embedded Methods (Built into model training)

Feature selection happens during model training — most efficient approach.

  • LASSO (L1 Regularisation): Shrinks unimportant feature weights to exactly zero
  • Decision Tree / Random Forest: Feature importance scores directly from splits
  • Elastic Net: Combination of L1 + L2 regularisation
⚡
Connection to Other Units

Feature selection (U4) directly feeds into supervised learning (U2) — better features → better Decision Tree, SVM, or Linear Regression. Also connects to PCA (U4) — but PCA is feature EXTRACTION (creates new features), while selection keeps existing ones.

Decision Tree — Algorithm (ID3)

A Decision Tree is a tree-structured classifier where each internal node represents a test on a feature, each branch represents an outcome of the test, and each leaf node represents a class label.

Core Formulas — Memorise BothEntropy H(S) = −Σ pᵢ log₂(pᵢ) [measure of impurity/disorder] → H(S) = 0 when all examples belong to one class (pure node) → H(S) = 1 when classes are equally split (maximum disorder) Information Gain IG(S, A) = H(S) − Σ [|Sᵥ|/|S|] × H(Sᵥ) → IG measures reduction in entropy after splitting on attribute A → ID3 selects the attribute with HIGHEST IG as the root/split node |Sᵥ| = number of examples with value v for attribute A · |S| = total examples
Simple Example — Play Tennis Dataset

Suppose we have 14 training examples: 9 play tennis (P), 5 don't (N). Features: Outlook (Sunny/Overcast/Rain), Temperature, Humidity, Windy.

Step-by-Step ID3
Step 1
Compute root entropy:
H(S) = −(9/14)log₂(9/14) − (5/14)log₂(5/14) = 0.94 bits
Step 2
Compute IG for each attribute (Outlook, Temperature, Humidity, Windy). The attribute with highest IG becomes the root node.
Step 3
Split on best attribute → create child nodes for each value → recursively apply on subsets.
Step 4
Stop when: node is pure (all one class) OR no more attributes OR all examples classified.
Algorithm Steps (Write in Exam)
  1. If all examples have same class → return leaf node with that class.
  2. If no features remaining → return majority class leaf.
  3. Compute entropy H(S) of current dataset S.
  4. For each feature A, compute Information Gain IG(S, A).
  5. Select feature A* with highest IG → make it the splitting node.
  6. For each value v of A*: create subtree recursively with subset S where A*=v.

Probabilistic Clustering — Core Idea

Unlike hard clustering (K-means assigns each point to exactly one cluster), probabilistic clustering assigns each data point a probability of belonging to each cluster. The most common model is the Gaussian Mixture Model (GMM).

Key difference from K-Means: K-means says "Point X belongs to Cluster 2." GMM says "Point X belongs to Cluster 2 with probability 0.7, Cluster 1 with probability 0.3."

Gaussian Mixture Model (GMM)

Assumes data is generated from K Gaussian distributions with unknown parameters (mean μₖ, covariance Σₖ, mixing weight πₖ).

GMM Formulap(x) = Σₖ πₖ × N(x; μₖ, Σₖ) Where: πₖ = mixing weight for cluster k (Σπₖ = 1) N(x; μₖ, Σₖ) = Gaussian PDF with mean μₖ and covariance Σₖ K = number of clusters Responsibility (soft assignment): rᵢₖ = P(cluster k | xᵢ) = πₖN(xᵢ;μₖ,Σₖ) / Σⱼ πⱼN(xᵢ;μⱼ,Σⱼ)
Expectation-Maximisation (EM) Algorithm — How GMM is Trained
  1. Initialise: Set initial values for μₖ, Σₖ, πₖ (often using K-means)
  2. E-Step (Expectation): Compute soft assignment probabilities rᵢₖ for each point i to cluster k
  3. M-Step (Maximisation): Update parameters: μₖ = Σrᵢₖxᵢ/Σrᵢₖ; Σₖ = weighted covariance; πₖ = mean rᵢₖ
  4. Repeat E and M steps until log-likelihood converges

Example: Heights of people from two populations (men: μ=175cm, women: μ=162cm). GMM can identify these two Gaussians from unlabelled height data. A person of height 168cm might get 0.4 probability of being male, 0.6 female.

What is the use of PCA in Machine Learning? Give the steps of PCA algorithm.

PCA — Uses in Machine Learning
  • Dimensionality Reduction: Reduces 100 features to 10 principal components while retaining ~95% variance. Reduces computation and memory.
  • Curse of Dimensionality: High dimensions cause sparsity — PCA compresses into a meaningful low-dimensional space.
  • Noise Reduction: Small eigenvalue components often correspond to noise — dropping them denoises data.
  • Visualisation: Reduce to 2D or 3D for plotting and human inspection of cluster structure.
  • Speed up training: Fewer features → faster ML algorithms (e.g., SVM, KNN).
  • Remove multicollinearity: PCA components are orthogonal (uncorrelated) — fixes issues in linear regression.
PCA Algorithm — Step-by-Step
PCA Steps (Memorise all 6)Step 1: Standardise the data → Subtract mean, divide by std: z = (x − μ)/σ for each feature Step 2: Compute Covariance Matrix → C = (1/n) × Xᵀ × X (n×n matrix for n features) Step 3: Compute Eigenvalues and Eigenvectors of C → C·v = λ·v (v = eigenvector, λ = eigenvalue) Step 4: Sort eigenvectors by eigenvalues in decreasing order → Largest eigenvalue = direction of maximum variance Step 5: Select top k eigenvectors (Principal Components) → k determined by "explained variance ratio" (e.g., keep 95% variance) Step 6: Project original data onto k components → Z = X × Wₖ (Wₖ = matrix of top k eigenvectors) Connection: SVD is an alternative to eigendecomposition for computing PCA — both give same result

Write short notes on: (a) Perceptions (b) Multilayer Network (c) Back Propagation (d) Artificial Neural Network

(a) Perceptron

A Perceptron is the simplest neural network unit — a mathematical model of a biological neuron. It takes multiple weighted inputs, sums them, adds a bias, and passes through an activation function.

Perceptron ModelOutput: ŷ = f(Σᵢ wᵢxᵢ + b) where f = activation function (step/sign) Perceptron Learning Rule (weight update): wᵢ ← wᵢ + α(y − ŷ)xᵢ where α = learning rate, y = actual, ŷ = predicted b ← b + α(y − ŷ) Limitation: Can only solve LINEARLY SEPARABLE problems (e.g., AND, OR — not XOR). This is the "XOR problem" that drove development of multilayer networks.
(b) Multilayer Network (Multilayer Perceptron — MLP)

A Multilayer Network extends the perceptron by adding one or more hidden layers between input and output, allowing it to learn non-linear decision boundaries and solve problems like XOR.

📊 MLP STRUCTURE (Draw this)
Input Layer (features) → Hidden Layer 1 (neurons + ReLU) → Hidden Layer 2 → Output Layer (Sigmoid/Softmax)
Each connection has a weight · Each neuron has a bias · Fully connected (Dense) layers

Why hidden layers? Each hidden layer learns progressively more abstract representations. Universal Approximation Theorem: a single hidden layer with enough neurons can approximate any continuous function.

(c) Backpropagation

Backpropagation (Backward Propagation of Errors) is the algorithm used to train MLPs. It efficiently computes gradients of the loss function with respect to all weights using the chain rule, then updates weights via gradient descent.

Backpropagation Steps1. Forward Pass: Compute ŷ by passing input through all layers 2. Compute Loss: L = loss(y, ŷ) e.g., MSE or Cross-Entropy 3. Backward Pass: Compute ∂L/∂w for each weight using chain rule: ∂L/∂wᵢⱼ = ∂L/∂aⱼ × ∂aⱼ/∂zⱼ × ∂zⱼ/∂wᵢⱼ (propagated backwards layer by layer) 4. Weight Update: wᵢⱼ ← wᵢⱼ − α × ∂L/∂wᵢⱼ (α = learning rate) Chain rule = the mathematical backbone of backprop. ∂L/∂w for output layer is easy; hidden layer gradients are computed by multiplying through the chain.
(d) Artificial Neural Network (ANN)

An ANN is a computational system loosely inspired by biological neural networks in the brain. It consists of interconnected nodes (artificial neurons) organised in layers, capable of learning complex patterns from data.

ComponentDescription
Input LayerReceives raw feature values — one neuron per feature
Hidden LayersTransform inputs into progressively abstract representations
Output LayerProduces final prediction (1 neuron for regression, K for K-class softmax)
Weights (w)Learnable parameters connecting neurons — adjusted by backprop
Bias (b)Constant offset per neuron — adds flexibility to the model
ActivationNon-linear function: ReLU, Sigmoid, Tanh, Softmax

Training: Initialise random weights → Forward pass → Compute loss → Backprop → Update weights → Repeat for epochs. Applications: Image recognition, NLP, speech synthesis, drug discovery, autonomous vehicles.

SARSA — Overview

SARSA (State-Action-Reward-State-Action) is an on-policy temporal difference (TD) learning algorithm for solving MDPs. It learns the Q-value (action-value) function Q(s, a) — the expected total reward of taking action a in state s and following the current policy thereafter.

The name comes from the tuple used in each update: (Sₜ, Aₜ, Rₜ₊₁, Sₜ₊₁, Aₜ₊₁) — the current state, action taken, reward received, next state, AND the next action chosen by the current policy.

SARSA Update RuleQ(Sₜ, Aₜ) ← Q(Sₜ, Aₜ) + α[Rₜ₊₁ + γQ(Sₜ₊₁, Aₜ₊₁) − Q(Sₜ, Aₜ)] Where: α = learning rate (0 < α ≤ 1) γ = discount factor (0 ≤ γ ≤ 1) Rₜ₊₁ = reward after taking action Aₜ in state Sₜ Q(Sₜ₊₁, Aₜ₊₁) = Q-value of next state-action pair (ACTUALLY taken by policy) Key: Aₜ₊₁ is sampled from the CURRENT policy π (e.g., ε-greedy) → ON-POLICY
SARSA Algorithm Steps
  1. Initialise Q(s, a) for all state-action pairs (often to 0)
  2. For each episode: observe initial state S
  3. Choose action A from S using policy π (e.g., ε-greedy from Q)
  4. Take action A → observe reward R and next state S'
  5. Choose A' from S' using same policy π
  6. Update: Q(S,A) ← Q(S,A) + α[R + γQ(S',A') − Q(S,A)]
  7. S ← S'; A ← A'; go to step 4 until terminal state

Advantages of Naive Bayes
  • Fast and efficient: Training O(n·d), Prediction O(d·K) — extremely fast, suitable for real-time classification.
  • Works well with small data: Doesn't need large datasets to give reasonable estimates (compared to SVM, neural nets).
  • Handles high dimensions naturally: Excellent for text classification (thousands of word features) — document spam filtering, news categorisation.
  • Easy to implement: Just compute frequencies and apply Bayes' theorem — no iterative optimisation needed.
  • Online learning: Model can be updated incrementally as new data arrives — just update counts.
  • Robust to irrelevant features: Each feature independently contributes — irrelevant features add near-zero evidence.
Algorithm Steps
  1. Training Phase — Learn from Data:
    • Compute P(Cₖ) = count(class Cₖ) / total samples (prior probability for each class)
    • For each feature xⱼ and class Cₖ: compute P(xⱼ = vᵢ | Cₖ) = count(xⱼ=vᵢ in class Cₖ) / count(class Cₖ)
    • Apply Laplace smoothing if any count is zero: add 1 to all counts
  2. Prediction Phase — Classify New Instance X = (x₁,...,xₙ):
    • For each class Cₖ: compute score = P(Cₖ) × Π P(xⱼ | Cₖ)
    • Predicted class = argmax_k [P(Cₖ) × Π P(xⱼ | Cₖ)]
    • (In practice, take log to avoid underflow: log P(Cₖ) + Σ log P(xⱼ | Cₖ))

Multilayer Neural Network (MLP)

A Multilayer Neural Network (also called Multilayer Perceptron or MLP) is an Artificial Neural Network with at least one hidden layer between the input and output layers. The "multiple layers" allow the network to learn non-linear mappings — solving problems that a single-layer perceptron cannot (like XOR).

📊 MLP ARCHITECTURE (Draw This — Essential)
Input Layer (x₁...xₙ) → Hidden Layer 1 (h₁...hₖ, activation=ReLU) → [Hidden Layer 2...] → Output Layer (ŷ, Sigmoid/Softmax)
Each arrow = a weight w · Each node = a neuron with bias b
Computation at each neuron: z = Σwᵢxᵢ + b → a = f(z) [f = activation function]
Forward Pass — Layer-by-Layer ComputationFor layer l with nl neurons and nl-1 inputs: Z[l] = W[l] × A[l-1] + b[l] (weighted sum + bias) A[l] = f(Z[l]) (apply activation function) Where: W[l] = weight matrix of layer l (shape: nl × nl-1) A[l-1] = activations from previous layer (or input X for l=1) b[l] = bias vector · f = activation function (ReLU, Sigmoid, etc.) Output layer (classification): A[L] = Softmax(Z[L]) → probability distribution Loss: L = CrossEntropy(y, A[L]) = −Σ yᵢ log(ŷᵢ)
Backpropagation Learning Algorithm

Backpropagation is the algorithm that trains MLPs by computing gradients of the loss function with respect to every weight in the network — using the chain rule of calculus — and then updating weights via gradient descent.

Intuition: If a neuron contributes to a large error at the output, backprop "blames" it proportionally and adjusts its weights more. The blame propagates backward through layers.

Backpropagation — Full AlgorithmPHASE 1: FORWARD PASS For l = 1, 2, ..., L: Z[l] = W[l]·A[l-1] + b[l] A[l] = f(Z[l]) Compute Loss: L = loss(y, A[L]) PHASE 2: BACKWARD PASS (compute gradients) Output layer error: δ[L] = ∂L/∂Z[L] = A[L] − y (for softmax + cross-entropy) Hidden layer error (propagate backwards from l=L-1 to l=1): δ[l] = (W[l+1]ᵀ · δ[l+1]) ⊙ f'(Z[l]) (⊙ = element-wise multiply · f' = derivative of activation) Gradients: ∂L/∂W[l] = δ[l] · A[l-1]ᵀ ∂L/∂b[l] = δ[l] PHASE 3: WEIGHT UPDATE (Gradient Descent) W[l] ← W[l] − α · ∂L/∂W[l] b[l] ← b[l] − α · ∂L/∂b[l] (α = learning rate, typically 0.001–0.1) REPEAT for all training examples (or mini-batches) until convergence
Why Chain Rule is Central to Backprop

The weight w in layer 1 affects A[1] → Z[2] → A[2] → ... → Loss. To find ∂L/∂w, we must chain through all these intermediates. Chain rule: ∂L/∂w = (∂L/∂A[L]) × (∂A[L]/∂Z[L]) × ... × (∂Z[2]/∂A[1]) × (∂A[1]/∂Z[1]) × (∂Z[1]/∂w). Backprop efficiently reuses already-computed δ values to avoid redundant work.

Activation Function Derivatives (Needed for Backprop)
Key DerivativesSigmoid: f(z) = 1/(1+e⁻ᶻ) → f'(z) = f(z)(1−f(z)) ReLU: f(z) = max(0,z) → f'(z) = 1 if z>0, else 0 Tanh: f(z) = tanh(z) → f'(z) = 1 − tanh²(z) Vanishing Gradient Problem: Sigmoid/Tanh saturate → f'(z)≈0 for large |z| → gradients die in deep networks. ReLU fixes this (f'=1 for positive z) — hence ReLU is preferred in hidden layers.

(a) Decision Tree Algorithm — 5 marks

A Decision Tree is a supervised learning algorithm that models decisions as a tree structure. Each internal node tests a feature, each branch represents an outcome, and each leaf node gives a class label (classification) or numeric value (regression).

Core Formula — ID3 AlgorithmEntropy: H(S) = −Σᵢ pᵢ log₂(pᵢ) → Measures impurity: 0 = pure node, 1 = 50-50 split (binary) Information Gain: IG(S, A) = H(S) − Σᵥ (|Sᵥ|/|S|) × H(Sᵥ) → Attribute with HIGHEST IG is chosen to split Gini Index (CART): Gini(S) = 1 − Σᵢ pᵢ² → Attribute with LOWEST Gini is chosen (0 = pure, 0.5 = equal split)
Algorithm Steps
  1. If all examples in S belong to same class → return leaf node (that class)
  2. If no attributes remain → return leaf (majority class)
  3. Compute IG (or Gini) for all attributes
  4. Select attribute A* with highest IG → create decision node
  5. For each value v of A*: recursively build subtree on subset {S where A*=v}
  6. Pruning (optional): remove branches that don't improve test accuracy

(a) Collaborative Filtering — 5 marks

Collaborative Filtering (CF) is a recommendation technique that predicts a user's preferences based on the preferences of similar users (or items), without requiring any knowledge of item content. The core assumption: users who agreed in the past will agree in the future.

Similarity Measures Used in CFCosine Similarity: sim(u, v) = (u · v) / (||u|| × ||v||) → Measures angle between user rating vectors Pearson Correlation: sim(u, v) = Σᵢ (rᵤᵢ − r̄ᵤ)(rᵥᵢ − r̄ᵥ) / [√Σ(rᵤᵢ−r̄ᵤ)² × √Σ(rᵥᵢ−r̄ᵥ)²] → Accounts for different rating scales between users Prediction for user u, item i: r̂ᵤᵢ = r̄ᵤ + [Σᵥ sim(u,v)(rᵥᵢ − r̄ᵥ)] / Σᵥ |sim(u,v)|

(b) Evaluating ML Algorithms and Model Selection — 5 marks
Why Evaluation Matters

Training accuracy is meaningless — a model that memorises training data (overfitting) appears perfect on training but fails on new data. Proper evaluation estimates generalisation performance — how well the model will work on unseen data.

Evaluation Methods
  1. Hold-Out (Train-Test Split): Typically 70-30 or 80-20 split. Simple but result depends on which data ends up in each split — high variance.
  2. K-Fold Cross-Validation: Split data into K equal folds. Train on K-1, test on 1. Repeat K times. Average accuracy = final metric. K=10 is most common. Much more reliable than hold-out.
  3. Leave-One-Out (LOO-CV): K-fold with K=n. Every single data point used as test once. Very thorough but computationally expensive for large n.
  4. Stratified K-Fold: Ensures each fold has the same class proportion as the original dataset — essential for imbalanced datasets.
Model Selection Criteria
Key Evaluation Metrics for Model SelectionClassification: Accuracy = (TP+TN)/N · Precision = TP/(TP+FP) · Recall = TP/(TP+FN) F1 = 2PR/(P+R) · AUC-ROC (plot TPR vs FPR across thresholds) Regression: MSE = (1/n)Σ(y−ŷ)² · RMSE = √MSE · MAE = (1/n)Σ|y−ŷ| · R² = 1−SSres/SStot Complexity Penalty (prefer simpler models): AIC = 2k − 2ln(L) k = number of parameters · L = likelihood BIC = k·ln(n) − 2ln(L) (penalises complexity more than AIC)

The "Naive" in Naive Bayes refers to the strong (naïve) assumption of conditional independence among all input features given the class label.

The Naive AssumptionP(X|C) = P(x₁|C) × P(x₂|C) × ... × P(xₙ|C) This assumes: knowing the class C, each feature xᵢ is independent of every other xⱼ. → In reality, features ARE correlated (e.g., age and salary), but we ignore this. → This simplification (naive assumption) makes computation tractable. Why it still works: Even though the probability estimates are wrong, the RANKING of class probabilities is often correct → correct classification.

The assumption is "naive" because it is almost NEVER true in real data. Yet Naive Bayes works surprisingly well — especially in text classification (spam filtering) where this simplification is acceptable despite feature correlations.

Motivation for Gaussian Mixture Model (GMM) — 4 marks

Limitation of K-Means: K-means assigns each point to exactly ONE cluster (hard assignment) and assumes clusters are spherical with equal size. This fails for:

  • Clusters of different sizes or densities
  • Elliptical (elongated) clusters
  • Points near cluster boundaries — forced into one cluster with full certainty

GMM Motivation: Real data often comes from mixtures of multiple populations (e.g., heights of men and women combined). GMM models this as a mixture of K Gaussian distributions and provides soft assignments — each point has a probability of belonging to each cluster.

GMM Key Formulap(x) = Σₖ₌₁ᴷ πₖ × N(x; μₖ, Σₖ) → Data is a weighted mixture of K Gaussians → πₖ = mixing weight (Σπₖ=1) → μₖ, Σₖ = mean and covariance of cluster k Trained by Expectation-Maximisation (EM): E-step: Compute soft assignments r_ik = P(cluster k | xᵢ) M-step: Update μₖ, Σₖ, πₖ using weighted averages GMM advantage: Can model ANY shape (covariance Σₖ can be diagonal, full, etc.)
Relationship Between K-Means and PCA — 4 marks

K-Means and PCA both reduce data complexity but in fundamentally different ways — yet they are mathematically related:

AspectK-MeansPCA
TypeClustering (unsupervised)Dimensionality reduction (unsupervised)
OutputCluster assignments per pointLow-dimensional projection of data
ReducesNumber of groups (assigns group labels)Number of features (creates new axes)

Mathematical Connection: Both are related to eigendecomposition. K-means can be seen as a discrete, non-linear version of PCA. Curse of Dimensionality — 4 marks

The Curse of Dimensionality refers to various phenomena that arise when analysing data in high-dimensional spaces. As the number of features (dimensions) increases, the data becomes increasingly sparse and many ML algorithms break down.

  • Data Sparsity: In high dimensions, data points are far apart. A dataset that's "dense" in 2D becomes extremely sparse in 100D — no point is close to any other.

  • Overfitting: More features → more parameters → overfitting with limited samples.
  • Computational cost: Training time grows polynomially or exponentially with dimensions.

Slack Variables in SVM — 4 marks

The hard-margin SVM requires all points to be correctly classified with margin ≥ 1. But real data is often not linearly separable — some overlap exists. Slack variables (ξᵢ ≥ 0) allow soft-margin SVM to tolerate a limited amount of misclassification.

Soft-Margin SVM with Slack VariablesHard margin constraint: yᵢ(w·xᵢ + b) ≥ 1 (no violations allowed) Soft margin constraint: yᵢ(w·xᵢ + b) ≥ 1 − ξᵢ (ξᵢ = slack variable) → ξᵢ = 0: point correctly classified with margin ≥ 1 (no violation) → 0 < ξᵢ ≤ 1: point inside margin but correctly classified → ξᵢ > 1: point misclassified (on wrong side of hyperplane) Objective (Soft-Margin SVM): Minimise: (1/2)||w||² + C · Σᵢ ξᵢ Subject to: yᵢ(w·xᵢ+b) ≥ 1 − ξᵢ and ξᵢ ≥ 0 C = Regularisation parameter: Large C → Small slack tolerance → Narrow margin, fewer violations → Overfit risk Small C → Large slack tolerance → Wide margin, more violations → Underfit risk

Why slack variables? They provide a mathematical mechanism to handle noisy, overlapping, or non-linearly separable data without completely abandoning the SVM formulation. The penalty C·Σξᵢ in the objective ensures we minimise the total violation while still maximising the margin.

Why Precision Alone Is Insufficient

Precision = TP/(TP+FP) measures: "Of all predicted positives, how many are actually positive?" A model can achieve 100% precision by simply making very few positive predictions — only predicting positive when it is absolutely certain. But this model would miss many actual positives (high FN).

Example: Cancer detection dataset: 100 patients, 10 have cancer. Model predicts cancer for ONLY 1 patient (the most obvious case). TP=1, FP=0, FN=9, TN=90. Precision = 1/1 = 100% — perfect! But Recall = 1/10 = 10% — it missed 9 cancer patients. This model is clinically useless despite perfect precision.

Why Recall Alone Is Insufficient

Recall = TP/(TP+FN) measures: "Of all actual positives, what fraction did we detect?" A model can achieve 100% recall by predicting EVERYTHING as positive — then no positive is missed. But this generates massive false alarms.

Example: Same dataset. Model predicts cancer for ALL 100 patients. TP=10, FP=90, FN=0, TN=0. Recall=100% — caught every cancer! But Precision=10/100=10% — 90 healthy patients wrongly alarmed. This model is also clinically useless.

The Precision-Recall Tradeoff

There is an inherent tradeoff: increasing the positive prediction threshold increases Precision but decreases Recall. Lowering the threshold increases Recall but decreases Precision. Neither alone captures full model quality.

F1-Score — Balancing BothF1 = 2 × (Precision × Recall) / (Precision + Recall) → Harmonic mean: penalises extreme imbalances between P and R → F1 = 0 if either P=0 or R=0 → F1 = 1 only if BOTH P=1 AND R=1 When to use F1: When both false positives and false negatives have significant cost When to use Recall only: Medical diagnosis (FN = missed disease = dangerous) When to use Precision only: Information retrieval (FP = irrelevant results = annoying)

MDP — Formal Definition

A Markov Decision Process (MDP) provides a formal framework for an agent making sequential decisions in an environment under uncertainty. It is characterised by the Markov Property: the future depends only on the current state, not the history.

MDP = (S, A, T, R, γ)S = Set of states A = Set of actions (may depend on state) T(s, a, s') = P(s'|s,a) = Transition function (probability of reaching s' from s with action a) R(s, a, s') = Reward received for transition (or R(s) or R(s,a)) γ ∈ [0,1] = Discount factor Markov Property: P(Sₜ₊₁|Sₜ,Aₜ,Sₜ₋₁,...) = P(Sₜ₊₁|Sₜ,Aₜ)

Q6
K-means and Gaussian mixtures both gave 5 clusters with same centers. Can three points assigned to different K-means clusters be in the same GMM cluster? Why or why not?
Unit 3 — Unsupervised LearningK-Means vs GMMSoft vs Hard Clustering
2 Marks +
✦ Model Answer

Yes, this is possible. K-means performs hard assignment — each point belongs to exactly one cluster (the closest centroid). GMM performs soft assignment — each point has a probability of belonging to every cluster.

Why: Even if two points are assigned to different K-means clusters (because one is closest to Centroid A and another to Centroid B), in GMM, they might both have the highest probability for the same cluster K if that Gaussian has a broader covariance (spread out ellipse) that captures both points. GMM's covariance matrix Σₖ allows elongated, non-spherical cluster shapes — K-means assumes spherical equal-variance clusters.

Example: Three points along a diagonal line. K-means splits them into 3 nearest-centroid groups. A GMM Gaussian with a diagonal covariance aligned to the line could assign all three the highest probability under the same component.

Naive Bayes applies Bayes' theorem with the naive (strong) assumption that ALL features are conditionally independent given the class label. This simplification is almost never true in real data — yet it leads to surprisingly good classification results.
