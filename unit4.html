Introduction to Statistical Learning Theory

75% · Foundation

What is Statistical Learning Theory?

Statistical Learning Theory (SLT) is the mathematical framework that answers: "When and why does machine learning work?" It studies the conditions under which a learning algorithm can generalize from training data to unseen data.

Core question: Given a training set of n examples, how much can we trust that a model's training accuracy reflects its true accuracy on new data? SLT gives us the theoretical tools (bounds, risk functions, VC dimension) to answer this.

Key Statistical Concepts (Part A Ready)

ConceptDefinitionWhy Important
Mean (μ)Average value: μ = (1/n)·ΣxᵢCentral tendency of data — model's expected output
Variance (σ²)σ² = (1/n)·Σ(xᵢ − μ)²Spread of data — model's inconsistency across datasets
Standard Deviation (σ)σ = √[Σ(xᵢ − μ)²/n]Typical deviation from mean — feature scaling, model stability
MSEMSE = (1/n)Σ(yᵢ − ŷᵢ)²Regression loss — measures prediction error

Standard Deviation in ML (2025 Part A Q1 + Q8)

Standard deviation tells us how spread out the data values are around the mean. In ML it is critical for:

  • Feature Scaling: Z-score normalization uses SD: z = (x − μ)/σ. Essential before KNN, SVM, PCA — they are distance-sensitive.
  • Detecting outliers: Points beyond 3σ from mean are typically outliers.
  • Measuring model stability: High SD in cross-validation scores → unstable model.
  • Understanding data spread: High SD → diverse dataset → model must generalize more broadly.
Variance and Standard Deviation — Formulas
Mean: μ = (1/n) · Σᵢ xᵢ
Variance: σ² = (1/n) · Σᵢ (xᵢ − μ)²
Standard Deviation: σ = √σ²
For a sample (not full population): use (n−1) denominator (Bessel's correction): s² = Σ(xᵢ − x̄)²/(n−1)
📌 Numerical — 2025 Part A Q8: Calculate variance and SD for {4, 2, 5, 8, 6}
Data: 4, 2, 5, 8, 6   (n=5)

Step 1: Mean
μ = (4+2+5+8+6)/5 = 25/5 = 5

Step 2: Deviations from mean
x₁−μ = 4−5 = −1  →  (x₁−μ)² = 1
x₂−μ = 2−5 = −3  →  (x₂−μ)² = 9
x₃−μ = 5−5 =  0  →  (x₃−μ)² = 0
x₄−μ = 8−5 =  3  →  (x₄−μ)² = 9
x₅−μ = 6−5 =  1  →  (x₅−μ)² = 1

Step 3: Variance
σ² = (1+9+0+9+1)/5 = 20/5 = 4

Step 4: Standard Deviation
σ = √4 = 2

Answer: Variance = 4, Standard Deviation = 2
Interpretation: Values typically deviate 2 units from the mean of 5.

Bias-Variance Tradeoff — The Central Theorem of SLT

The expected prediction error of any model can be decomposed into three parts:

Bias-Variance Decomposition (★ MUST KNOW ★)
Expected Error = Bias² + Variance + Irreducible Noise
E[(y − ŷ)²] = [E[ŷ] − y]² + E[(ŷ − E[ŷ])²] + σ²_noise
Bias = error from wrong assumptions (underfitting). Variance = error from sensitivity to training data (overfitting). Noise = irreducible error in the data itself.
AspectHigh BiasHigh Variance
Model typeToo simple (underfitting)Too complex (overfitting)
Training errorHighLow
Test errorHighHigh
ExampleLinear model for non-linear dataDeep unpruned decision tree
SolutionMore complex model, more featuresRegularization, more data, pruning

Connection to Unit 2 — Decision Trees

Decision Trees (Unit 2) are high-variance models — a small change in training data can produce a completely different tree. Random Forest (Unit 2) reduces variance by averaging many trees. SVM is a low-variance model. This is why Unit 4's bias-variance theory directly explains why ensemble methods (Unit 2) work: they trade a tiny bit of bias for a large reduction in variance.

Curse of Dimensionality

The curse of dimensionality refers to phenomena that arise when analyzing data in high-dimensional spaces. As the number of features increases, the data becomes increasingly sparse, making learning harder.

  • Exponential data requirement: To maintain density, training data must grow exponentially with dimensions. With 10 features and 10 values each, you need 10¹⁰ examples for complete coverage.
  • Distance loses meaning: In high dimensions, all pairwise distances converge to the same value — nearest neighbor becomes meaningless.
  • Overfitting increases: More features = more model parameters = higher overfitting risk.
  • Computational cost: O(d²) or O(d³) algorithms (like PCA) become expensive as d grows.

Solution: Dimensionality Reduction

PCA, SVD, and feature selection (all in Unit 4) address the curse of dimensionality by reducing the number of features while preserving most information.

Principal Component Analysis (PCA)

🔥 90% — Asked Every Year

What is PCA? (From Scratch)

PCA (Principal Component Analysis) is an unsupervised, linear dimensionality reduction technique that transforms a dataset with many features into a smaller set of uncorrelated features called Principal Components (PCs), while preserving as much variance (information) as possible.

Intuition: You have photos of a 3D object taken from thousands of angles. Most angles contain redundant information. PCA finds the 2–3 "best viewing angles" that capture the most variation. These are the principal components — they explain the data with far fewer dimensions.

When to use PCA: When features are highly correlated, when you need to reduce computational cost, when you want to visualize high-dimensional data in 2D/3D, and as preprocessing before KNN, SVM, or neural networks.

Fig 2.1 — PCA projects data onto PC1 (direction of maximum variance), reducing from 2D to 1D while keeping most information.

PCA Algorithm — Full Steps (★ Write All 6 Steps ★)

1
Standardize the data: For each feature, compute Z-score: xᵢ_norm = (xᵢ − μᵢ)/σᵢ. This ensures all features have mean=0, SD=1. Essential — PCA is sensitive to scale.
2
Compute Covariance Matrix: C = (1/(n−1)) · XᵀX where X is the mean-centered data matrix. The (i,j) entry = covariance between feature i and j.
3
Compute Eigenvalues and Eigenvectors of the covariance matrix C: C·v = λ·v where λ = eigenvalue, v = eigenvector (principal direction).
4
Sort by Eigenvalue: Rank eigenvectors by their eigenvalues in descending order. Largest eigenvalue → PC1 (explains most variance), second largest → PC2, etc.
5
Select k components: Choose top k eigenvectors (components) that explain ≥ 95% of total variance. Form matrix W = [v₁ | v₂ | ... | vₖ] (d×k).
6
Project the data: Z = X · W. Now Z is the reduced dataset with k features instead of d (d >> k).
PCA — Key Formulas (★ All Must Be in Your Answer ★)
Covariance Matrix: C = (1/n−1) · XᵀX (d×d matrix)
Eigendecomposition: C = VΛVᵀ where Λ = diagonal matrix of eigenvalues
Variance explained by PCᵢ = λᵢ / Σλⱼ × 100%
Projection: Z = X · V_k (n×d matrix → n×k matrix)
The eigenvectors (principal components) are orthogonal to each other — they capture independent directions of variation in the data.

Scree Plot — Choosing k

A scree plot shows eigenvalues in decreasing order. Choose k where the curve "elbows" — beyond that point, additional components explain very little variance. Common rule: keep components until cumulative variance ≥ 95%.

PCA Example — 2D to 1D

📌 Simple PCA Worked Example
Original data (3 points, 2 features: Height, Weight):
Point 1: (170, 65)
Point 2: (180, 75)
Point 3: (160, 55)

Step 1: Compute means
μ_height = (170+180+160)/3 = 170
μ_weight = (65+75+55)/3 = 65

Step 2: Center the data (subtract mean)
X_centered:
Point 1: (170−170, 65−65) = (0, 0)
Point 2: (180−170, 75−65) = (10, 10)
Point 3: (160−170, 55−65) = (−10, −10)

Step 3: Covariance matrix (2×2)
C = (1/(n−1)) · XᵀX
Σx² = 0+100+100 = 200, Σy² = 0+100+100 = 200, Σxy = 0+100+100 = 200
C = (1/2) × [[200, 200], [200, 200]] = [[100, 100], [100, 100]]

Step 4: Eigenvalues
det(C − λI) = 0
(100−λ)² − 100² = 0  → (100−λ)² = 100² → 100−λ = ±100
λ₁ = 200 (PC1), λ₂ = 0 (PC2)

Step 5: Eigenvectors
For λ₁=200: v₁ = [1/√2, 1/√2] ≈ [0.707, 0.707]
For λ₂=0:   v₂ = [1/√2, −1/√2]

Step 6: PC1 explains λ₁/(λ₁+λ₂) = 200/200 = 100% of variance!
→ Data is perfectly 1D along the (Height+Weight) diagonal.

Projection onto PC1:
z₁ = 0×0.707 + 0×0.707 = 0
z₂ = 10×0.707 + 10×0.707 = 14.14
z₃ = −10×0.707 + (−10)×0.707 = −14.14

Result: 2D data → 1D with zero information loss!
⚡
PCA Exam Validation Trick

Always verify: (1) Number of principal components ≤ min(n, d). (2) Sum of all eigenvalues = trace of covariance matrix = sum of feature variances. (3) PC1 variance explained should be the highest. (4) All principal components are orthogonal — dot product of any two = 0. If your eigenvectors v₁ and v₂ satisfy v₁·v₂ ≠ 0, you made an error.

PCA Applications (RTU Filler Theory)

  • Face recognition: Eigenfaces — each face is a combination of principal "face components"
  • Recommendation systems: SVD (related to PCA) used in Netflix-style recommenders
  • Genomics: Reducing thousands of gene expression features to 10–20 PCs
  • Image compression: Store only top-k principal components instead of all pixels
  • Finance: Finding key risk factors from hundreds of correlated stock prices
🔗
Connection — PCA and K-Means (Unit 3)

PCA and K-means both deal with unlabelled data and find structure. PCA finds directions of maximum variance (global structure). K-means finds cluster centers (local structure). A common pipeline: apply PCA first to reduce dimensions, then run K-means on the reduced data. The 2023 exam question asks directly: "What is the relationship between K-means and PCA?" Answer: Both can be viewed as matrix factorization problems. PCA projects data to minimize reconstruction error; K-means assigns cluster labels to minimize within-cluster variance. Running K-means on PCA-reduced data is more efficient and often more accurate.

Q. What is the motivation behind Gaussian Mixture Model? What is the relationship between k-means and PCA?
Aug 2023 · Part B Q3 · 4+4 marks

Relationship between K-Means and PCA:

Both K-means and PCA can be viewed as forms of matrix factorization / data compression, but they work differently:

Similarities:
• Both are unsupervised — no labels required
• Both aim to find a compact representation of data
• Both can be viewed as approximating the data covariance structure

Differences:
• PCA finds continuous linear projections (principal axes) that maximize variance
• K-means finds discrete cluster assignments that minimize within-cluster variance
• PCA output: coordinates in PC space (real numbers)
• K-means output: cluster labels (integers)

Connection in practice:
Running K-means on PCA-reduced data is a very common pipeline: PCA first removes noise and correlation, then K-means finds cleaner clusters in the reduced space. This is called "spectral clustering" when eigendecomposition of the similarity matrix is used instead of covariance matrix.

Mathematically: K-means finds cluster means μₖ such that Σᵢ ||xᵢ − μₖᵢ||² is minimized. PCA finds projection W such that ||X − XWWᵀ||² (reconstruction error) is minimized. Both minimize a squared distance objective.

Singular Value Decomposition (SVD)

75% · Part A/B

What is SVD?

Singular Value Decomposition (SVD) is a matrix factorization technique that decomposes any real matrix A into three matrices. It is the generalization of eigendecomposition to non-square matrices and is the mathematical foundation of PCA.

Intuition: SVD is like discovering the "skeleton" of a matrix — the most important structural components. You can discard the less important parts and still reconstruct a good approximation of the original.

SVD Decomposition Formula (★ MUST MEMORIZE ★)
A = U · Σ · Vᵀ
where: A is an m×n matrix
U = m×m orthogonal matrix (left singular vectors — column patterns)
Σ = m×n diagonal matrix (singular values σ₁ ≥ σ₂ ≥ ... ≥ σᵣ ≥ 0)
Vᵀ = n×n orthogonal matrix (right singular vectors — row patterns)
U and V columns are orthonormal. Σ is diagonal with non-negative singular values on diagonal.

Relationship between SVD and PCA

If X is mean-centered (zero mean), then:
XᵀX = V · Σ² · Vᵀ (eigendecomposition of covariance matrix)
PCA principal components = right singular vectors V of X
PCA eigenvalues = σᵢ² (squares of singular values)
PCA = SVD applied to the mean-centered data matrix. SVD is more numerically stable than computing eigenvalues of XᵀX directly.

Truncated SVD — Low-Rank Approximation

Instead of using all singular values, we keep only the top k largest ones. This gives the best rank-k approximation of A:

Low-Rank Approximation (Truncated SVD)
Aₖ = U_k · Σ_k · Vₖᵀ
where U_k = first k columns of U, Σ_k = top k×k block, Vₖᵀ = first k rows of Vᵀ
By Eckart-Young theorem: Aₖ is the best rank-k approximation of A in terms of Frobenius norm.

Feature Selection Methods

80% · Part A + B

What is Feature Selection?

Feature Selection is the process of selecting a subset of relevant features from the original feature set. Unlike PCA (which transforms features), feature selection keeps the original features — just fewer of them.

Why do it: Removing irrelevant/redundant features reduces overfitting, speeds up training, improves interpretability, and avoids the curse of dimensionality.

Feature Ranking

Before selecting features, we rank them by some importance score. Common ranking criteria:

  • Correlation with target: Pearson correlation |corr(xᵢ, y)| — higher = more relevant
  • Information Gain: IG(xᵢ, y) from decision tree splitting (Unit 2 connection!)
  • Chi-squared test: χ² test for categorical features — measures dependence on class
  • Mutual Information: I(X;Y) = H(Y) − H(Y|X) — how much knowing X reduces uncertainty about Y

Three Main Feature Selection Categories (★ RTU Favourite ★)

Fig 4.1 — Three categories of Feature Selection Methods

1. Filter Methods

Filter methods rank features using statistical scores independent of any ML model. They are fast and scalable but don't consider feature interactions.

  • Pearson Correlation: Select features with |corr| > threshold. For regression. Misses non-linear relationships.
  • Chi-squared (χ²) test: For classification with categorical features. Tests if feature and class are independent. Higher χ² = more relevant.
  • Information Gain / Mutual Information: IG(X,Y) = H(Y) − H(Y|X). Used in decision trees as splitting criterion.
  • Variance threshold: Remove features with variance below threshold — low variance = low information.
  • ANOVA F-score: For numerical features in classification. Tests mean difference across classes.

2. Wrapper Methods

Wrapper methods use an actual ML model to evaluate feature subsets. They search over different combinations of features and pick the subset that gives the best cross-validation accuracy. Slow but typically gives best results.

  • Forward Selection: Start with no features. Add one feature at a time (the one that improves accuracy most). Stop when no improvement.
  • Backward Elimination: Start with all features. Remove one at a time (the least important). Stop when accuracy drops.
  • Recursive Feature Elimination (RFE): Train model → rank features by importance → remove weakest → repeat. Used with SVM, linear models.
  • Exhaustive Search: Try all 2ᵈ subsets → find optimal. Computationally infeasible for large d.

3. Embedded Methods

Embedded methods perform feature selection as part of the model training process itself. They combine the benefits of filter (speed) and wrapper (model-specific accuracy).

  • LASSO (L1 Regularization): Adds λ·Σ|βᵢ| penalty to regression loss. Forces irrelevant feature weights to exactly zero → automatic selection.
  • Ridge (L2 Regularization): Adds λ·Σβᵢ² penalty. Shrinks weights but doesn't zero them (not true selection — more of regularization).
  • Decision Tree / Random Forest Importance: Feature importance = total reduction in Gini/entropy from that feature across all splits and all trees.
  • Elastic Net: Combines L1 + L2: λ₁·Σ|βᵢ| + λ₂·Σβᵢ². Best of both.
LASSO vs Ridge vs Elastic Net
Ridge (L2): minimize Σ(yᵢ−ŷᵢ)² + λ·Σβⱼ² → shrinks weights, no zeroing
LASSO (L1): minimize Σ(yᵢ−ŷᵢ)² + λ·Σ|βⱼ| → zeros out weak features
Elastic Net: minimize Σ(yᵢ−ŷᵢ)² + λ₁·Σ|βⱼ| + λ₂·Σβⱼ² → best of both
LASSO is feature selection. Ridge is regularization. Use LASSO when you expect few relevant features. Use Ridge when you expect many small effects.

2. Wrapper Methods (model-dependent, slow):
Search over feature subsets by training and evaluating a model on each subset. Choose subset with best cross-validation accuracy.
Types: Forward selection (add one at a time), Backward elimination (remove one at a time), RFE (Recursive Feature Elimination).
Example: RFE with SVM — train SVM, rank features by weight magnitude, remove smallest, repeat.

3. Embedded Methods (done during training):
Feature selection is built into the model's learning process.
LASSO regression adds L1 penalty (λΣ|βᵢ|) — drives irrelevant feature weights to zero.
Random Forest provides feature_importance scores based on Gini reduction.

Filter is fastest but ignores interactions. Wrapper is most accurate but slow. Embedded balances both.
Q. Define attribute selection measure in classification.
2024 · Part A Q6 · 2 marks
Topic: Attribute Selection Measure · Unit 2/4
Attribute Selection Measure (ASM) is a heuristic used in decision tree algorithms to determine the best attribute (feature) for splitting a node, maximizing data homogeneity in child nodes.

Two main measures:
1. Information Gain (ID3/C4.5): IG(S,A) = H(S) − Σᵥ(|Sᵥ|/|S|)·H(Sᵥ). Select attribute with max IG. Biased towards attributes with many values.
2. Gini Index (CART): Gini(S) = 1 − Σpᵢ². Select attribute with minimum weighted Gini. Favors binary splits.

These measures guide which feature is used at each tree node — essentially a filter-type feature selection integrated into the Decision Tree algorithm.

The Confusion Matrix — Foundation of All Metrics

A Confusion Matrix is a 2×2 table (for binary classification) that breaks down predictions into four categories:

Fig 5.1 — Confusion Matrix. TP and TN are correct predictions; FP and FN are errors.
All Evaluation Metrics from Confusion Matrix (★★★ MEMORIZE ALL ★★★)
Accuracy = (TP + TN) / (TP + TN + FP + FN)
Precision = TP / (TP + FP) ← "Of predicted +, how many are actually +?"
Recall = TP / (TP + FN) ← "Of actual +, how many did we catch?"
F1-Score = 2 × (P × R) / (P + R) ← Harmonic mean of Precision and Recall
Specificity = TN / (TN + FP) ← True Negative Rate
FPR = FP / (FP + TN) ← False Positive Rate (x-axis of ROC)
Type I Error = FP (False Positive). Type II Error = FN (False Negative). In medical diagnosis, FN (missed cancer) is more dangerous than FP.

Why Precision/Recall Alone is Insufficient (2023 Part C Q2)

This is a critical 15-mark question from 2023. The answer requires explaining the precision-recall tradeoff and then AUC-ROC.

Problem with Accuracy alone: With class imbalance (99% negative, 1% positive), a model that always predicts "negative" achieves 99% accuracy but is useless (0% recall).

Problem with Precision alone: A spam filter that only flags 1 email as spam (and it's correct) achieves 100% precision, but misses 99 actual spam emails (terrible recall).

Problem with Recall alone: A spam filter that flags ALL emails as spam achieves 100% recall, but flags everything (terrible precision).

Solution — Precision-Recall Tradeoff: Increasing the classification threshold increases precision but decreases recall (and vice versa). We need a single metric that balances both → F1-Score or AUC-ROC.

AUC-ROC Curve — Full Explanation

ROC (Receiver Operating Characteristic) curve plots True Positive Rate (Recall) vs. False Positive Rate at various classification thresholds. AUC (Area Under Curve) is the area under this curve.

Fig 5.2 — ROC Curve. Each point = one threshold setting. AUC = area under violet curve. Higher AUC = better classifier.
AUC-ROC Key Facts
AUC = 1.0 → Perfect classifier (impossible in practice)
AUC = 0.5 → Random classifier (diagonal line = chance)
AUC < 0.5 → Worse than random (reverse labels to fix)
AUC ≥ 0.9 → Excellent. 0.8–0.9 = Good. 0.7–0.8 = Acceptable.
AUC is threshold-independent — it measures classifier performance across ALL possible thresholds. This makes it ideal for comparing classifiers regardless of operating point.

MSE — Mean Squared Error (2025 Part A Q9)

Regression Metrics
MSE = (1/n) · Σᵢ (yᵢ − ŷᵢ)² ← Penalizes large errors heavily
RMSE = √MSE ← Same units as target variable
MAE = (1/n) · Σᵢ |yᵢ − ŷᵢ| ← Robust to outliers

Q. Explain why precision or recall alone is not a sufficient measure to measure classifier performance in binary classification. Discuss AUC-ROC curve used in classification problems.
Aug 2023 · Part C Q2 · 15 marks
Topic: Precision/Recall, AUC-ROC · Unit 4
Why Precision or Recall Alone is Insufficient:

In binary classification, the performance of a classifier depends on the threshold used to convert probability scores into class labels. Changing this threshold creates a tradeoff between precision and recall:

Precision alone is insufficient: A classifier can achieve 100% precision by predicting "positive" only for the one example it is most confident about. If that one prediction is correct, precision=1.0. But all other positives are missed (recall→0). This is useless in practice.

Recall alone is insufficient: A classifier can achieve 100% recall by predicting "positive" for every example. All actual positives are caught, but so are all negatives — precision collapses to (# positives/total).

The Precision-Recall Tradeoff:
• Lower threshold → more predicted positive → higher Recall, lower Precision
• Higher threshold → fewer predicted positive → higher Precision, lower Recall
Neither extreme is acceptable. F1-Score = 2PR/(P+R) balances both, but it's a single-threshold measure.

AUC-ROC Curve:
The ROC (Receiver Operating Characteristic) curve plots TPR (=Recall) vs FPR at every possible classification threshold (0 to 1). Each point on the curve is one threshold setting.

TPR = TP/(TP+FN) = Recall (y-axis)
FPR = FP/(FP+TN) = 1−Specificity (x-axis)

AUC (Area Under Curve) = probability that the classifier ranks a random positive higher than a random negative.
• AUC = 1.0: Perfect. • AUC = 0.5: Random. • AUC > 0.5: Better than random.

Advantages of AUC-ROC:
1. Threshold-independent — measures performance across ALL operating points
2. Scale-invariant — not affected by class imbalance
3. Ideal for comparing multiple classifiers

Q. Precision and recall are two essential quality measures. Explain why it is the usual practice to trade one measure for the other. Explain why the F-score is a good measure.
June 2022 · Part B Q4b · 8 marks
Topic: Precision-Recall Tradeoff, F1-Score · Unit 4
Trading Precision for Recall (and vice versa):
Both precision and recall depend on the classification threshold θ. When we change θ:
• Increase θ → fewer positive predictions → higher Precision, lower Recall
• Decrease θ → more positive predictions → lower Precision, higher Recall

This inherent tradeoff means we cannot simultaneously maximize both. The optimal tradeoff point depends on the application:
• Medical diagnosis (missed cancer deadly → minimize FN): prefer high Recall even at cost of Precision
• Spam filtering (important email lost → minimize FP): prefer high Precision even at cost of Recall

Why F-Score is a Good Measure:
F1-Score = 2 × (Precision × Recall) / (Precision + Recall)

F1 is the harmonic mean of Precision and Recall. Properties:
1. Penalizes imbalance: Unlike arithmetic mean, harmonic mean is pulled down by low values. If either P or R is near zero, F1 is near zero, even if the other is perfect. Example: P=1.0, R=0.01 → Arithmetic mean=0.505 (misleadingly high), Harmonic mean (F1)=0.02 (correctly low).
2. Single threshold metric: F1 gives one number at the operating threshold.
3. Handles imbalanced data better than accuracy.
Generalized Fβ: Fβ = (1+β²)·PR / (β²·P + R). β>1 → weights Recall more. β<1 → weights Precision more.
Q. Write short notes on: (b) Evaluating machine learning algorithms and model selection
2024 · Part C Q4b · 5 marks
Topic: ML Algorithm Evaluation + Model Selection · Unit 4
Evaluating ML Algorithms:

After training, we must estimate how well the model will perform on unseen data. Key methods:

1. Hold-out Validation: Split data into 70% train / 30% test. Train on training set, evaluate on test set. Simple but high variance estimate.

2. k-Fold Cross Validation: Split data into k equal folds. Train on k−1 folds, test on remaining fold. Repeat k times. Average scores give more reliable estimate.
Standard: k=10 (10-fold CV). Less data waste than hold-out.

3. Leave-One-Out CV (LOOCV): k=n case. Each sample is test set once. Unbiased but expensive O(n²).

4. Bootstrap: Sample n examples WITH replacement. Train on bootstrap sample, test on out-of-bag samples.

Model Selection:
Model selection = choosing the best algorithm AND hyperparameters for a given problem.

Steps:
1. Define search space: which algorithms and hyperparameters to try
2. Use cross-validation to estimate performance of each configuration
3. Select configuration with best validation score
4. Retrain on all training data with selected configuration
5. Report final performance on held-out test set (used ONCE at the very end)

Techniques: Grid Search (exhaustive), Random Search (random sampling), Bayesian Optimization.

Warning: Never use test set for model selection — it causes optimistic bias (overfitting to test set).

Cross Validation — Core Concept

Cross Validation (CV) is a resampling technique to estimate model performance on unseen data using only the training set. It gives a less biased estimate than a single train/test split.

k-Fold Cross Validation
1. Split dataset D into k equal folds: D = D₁ ∪ D₂ ∪ ... ∪ Dₖ
2. For i = 1 to k: Train on D \ Dᵢ, Test on Dᵢ → get score sᵢ
3. CV score = (1/k) · Σᵢ sᵢ (average over k folds)
Standard choice: k=10. Stratified k-fold: maintains class proportions in each fold (important for imbalanced data).
Fig 6.1 — 5-Fold Cross Validation. Each fold takes a turn as the test set. Final score = average of 5 test scores.
