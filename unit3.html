Unsupervised Learning — Introduction

75%

What is Unsupervised Learning?

Unsupervised learning finds hidden patterns or structures in data WITHOUT labelled examples. There is no "correct answer" provided — the algorithm must discover structure on its own.

Contrast with supervised: In Unit 2 (Supervised), you trained models on (input, label) pairs. Here, you only have inputs — no labels. The algorithm must self-organise.

AspectSupervised (Unit 2)Unsupervised (Unit 3)
DataLabelled (x, y)Unlabelled (x only)
GoalLearn f(x) → yFind hidden structure
FeedbackLoss / error signalNone — intrinsic criteria
ExamplesDecision Tree, SVMK-Means, Apriori, GMM

Two Main Tasks

  • Clustering: Group similar data points together. (K-Means, Hierarchical, GMM)
  • Association Rule Mining: Find frequent patterns and rules. (Apriori, FP-Growth)

Requirements of a Good Clustering Algorithm (2024 Part A Q7)

  • Scalability: Handle large datasets efficiently
  • Ability to handle different data types: Numerical, categorical, text
  • Minimal domain knowledge: Should work with few parameters
  • Interpretability: Results should be understandable
  • Noise/Outlier handling: Robust to noisy data points
  • Insensitive to input order: Same result regardless of data order

K-Means Clustering

🔥 95% — Numerical Every Year!

Core Idea — Find k Natural Groups

K-Means partitions n data points into k clusters where each point belongs to the cluster with the nearest centroid. It minimizes the Within-Cluster Sum of Squares (WCSS) — the total distance from each point to its cluster centre.

Analogy: Imagine n students standing in a field. Pick k random "leaders." Each student walks to the nearest leader. Each leader moves to the centre of their group. Repeat until no one moves. That's K-Means.

K-Means Algorithm (Must Memorize)

1
Initialization: Choose k initial centroids (either randomly or using first k points or K-Means++ method).
2
Assignment Step: Assign each data point to the nearest centroid using Euclidean distance: d(x,μ) = √[(x₁-μ₁)² + (x₂-μ₂)²]
3
Update Step: Recalculate the centroid of each cluster as the mean of all points assigned to it: μₖ = (1/|Cₖ|) Σ xᵢ
4
Convergence Check: If centroids don't change (or change is below threshold), STOP. Otherwise go to Step 2.
Euclidean Distance Formula (★ Used in Every Numerical)
d(p, q) = √[(p₁ − q₁)² + (p₂ − q₂)²]
For new centroid: μ = (Σxᵢ/n, Σyᵢ/n) = (mean of x-coords, mean of y-coords)
⚡
Exam Calculation Trick

For Euclidean distance in RTU numericals, you DON'T need to take the square root when COMPARING distances — comparing d² is sufficient! This saves time and avoids √ errors. Example: d(P,C1)²= (2-5)²+(3-7)²=9+16=25. d(P,C2)²=(2-1)²+(3-2)²=1+1=2. Since 2 < 25, assign to C2. Always works!

Q. Suppose that you are to allocate several ATMs in a region. How can a clustering algorithm such as K-means be modified for quality clustering under both constraints (obstacle objects and minimum households)?
June 2022 · Part B Q3b · 8 marks
Topic: K-Means Application · Unit 3
Modified K-Means for ATM Allocation:

Standard K-Means modification needed:
The basic K-Means uses Euclidean distance and ignores real-world constraints. For ATM allocation with (i) obstacle constraints and (ii) minimum household constraints, we need:

1. Handling Obstacle Objects (bridges, rivers, highways):
Replace Euclidean distance with obstacle distance (shortest path avoiding obstacles). Use graph-based shortest path (Dijkstra's algorithm) to compute actual travel distance from household to ATM location. Standard K-Means: d(p,q) = Euclidean. Modified: d(p,q) = shortest path avoiding obstacles.

2. Handling Minimum Household Constraint (≥10,000 households per ATM):
Add a constraint to the assignment step: Each cluster must contain at least 10,000 households. If a cluster is too small, merge it with the nearest under-serviced cluster. Use constrained K-Means where cluster sizes have lower bounds.

Modified Algorithm:
Step 1: Initialize k ATM locations (centroids) considering obstacle-free locations
Step 2: Assign each household to nearest ATM using obstacle-aware distance
Step 3: Check constraint — if any cluster < 10,000 households, merge or reassign
Step 4: Update centroid = obstacle-free point minimizing total travel distance
Step 5: Repeat until convergence

This is called Constrained K-Means with Obstacle-Aware Distance Metric.

Hierarchical Clustering

85% · Part A/B

What is Hierarchical Clustering?

Hierarchical Clustering builds a tree of clusters (called a dendrogram) by either successively merging small clusters into larger ones (agglomerative / bottom-up) or splitting large clusters into smaller ones (divisive / top-down).

Key advantage over K-Means: No need to specify k in advance. You can choose any level of the dendrogram to get any number of clusters.

Fig 3.1 — Dendrogram: cutting at different heights gives different numbers of clusters

Agglomerative (Bottom-Up) Algorithm

1
Start: each data point is its own cluster (n clusters).
2
Compute pairwise distances between all clusters.
3
Merge the two clusters with the smallest distance.
4
Update distances between new cluster and all others.
5
Repeat until all points are in one cluster (root of dendrogram).

Inter-Cluster Similarity Measures (2024 Part B Q5 — ⭐ Very Important)

MethodFormulaProperty
Single Linkage (min)d(Ci,Cj) = min{d(p,q) | p∈Ci, q∈Cj}Minimum distance between clusters. Can create "chain" clusters (elongated). Example: two clusters connected by a single bridge point.
Complete Linkage (max)d(Ci,Cj) = max{d(p,q) | p∈Ci, q∈Cj}Maximum distance. Creates compact, roughly equal-size clusters. More robust to outliers.
Average Linkaged(Ci,Cj) = Σd(p,q)/(|Ci|×|Cj|)Average of all pairwise distances. Balance between single and complete.
Centroid Linkaged(Ci,Cj) = d(μi, μj)Distance between cluster means (centroids).
Ward's MethodMerges clusters with minimum increase in WCSSBest overall cluster quality. Tends to create equal-sized clusters.

Advantages & Disadvantages of Hierarchical Clustering (2025 Part A Q6)

AdvantagesDisadvantages
No need to specify k in advanceComputationally expensive O(n²logn) or O(n³)
Dendrogram shows full cluster hierarchyCannot undo a merge/split decision
Works with any distance measureSensitive to noise and outliers
Can discover clusters of arbitrary shapeDifferent linkage criteria give very different results
Deterministic — same result every runNot suitable for very large datasets
📝 RTU PYQs — Hierarchical Clustering
Q. What is a dendrogram in hierarchical clustering?
May 2025 · Part A Q7 · 2 marks
Topic: Hierarchical Clustering · Unit 3
A dendrogram is a tree-like diagram that shows the hierarchical relationship between clusters in hierarchical clustering. The x-axis represents individual data points and the y-axis represents the distance (or dissimilarity) at which clusters are merged. Cutting the dendrogram at any height gives a specific number of clusters. The longer the vertical line before a merge, the more dissimilar those clusters are. It allows visualizing the entire clustering process in one diagram and choosing the optimal number of clusters by looking for the "longest gap."
Q. How to define inter-cluster similarity in Hierarchical clustering? Give example of each approach.
2024 · Part B Q5 · 4 marks
Topic: Inter-Cluster Similarity · Unit 3
Inter-cluster similarity measures how far apart two clusters are. Four main approaches:

1. Single Linkage (MIN): d(Ci,Cj) = minimum distance between any point in Ci and any point in Cj.
Example: Clusters {A(1,1), B(2,2)} and {C(5,5), D(6,6)}. Single linkage = d(B,C) = √(9+9) ≈ 4.24.

2. Complete Linkage (MAX): d(Ci,Cj) = maximum distance between any point in Ci and any point in Cj.
Example: Same clusters. Complete linkage = d(A,D) = √(25+25) ≈ 7.07.

3. Average Linkage: d(Ci,Cj) = average of all pairwise distances between points in Ci and Cj.
Example: Average of d(A,C), d(A,D), d(B,C), d(B,D) = (5.66+7.07+4.24+5.66)/4 ≈ 5.66.

4. Centroid Linkage: d(Ci,Cj) = distance between centroids of Ci and Cj.
Example: centroid of Ci=(1.5,1.5), centroid of Cj=(5.5,5.5). Distance = √(16+16) ≈ 5.66.

Ward's Method: Merge clusters that minimize the total increase in WCSS. Preferred for balanced, compact clusters.

Probabilistic Clustering

75% · Part B

Hard vs Soft Clustering

In K-Means, each point belongs to exactly one cluster — this is called hard (crisp) clustering. But in reality, some points naturally lie between clusters. Probabilistic (soft) clustering assigns each point a probability of belonging to each cluster.

AspectHard Clustering (K-Means)Soft/Probabilistic Clustering
MembershipBinary (0 or 1)Probability [0,1] for each cluster
Point between clustersForced into oneGets partial membership
UncertaintyCannot representNaturally represented
Example algorithmK-MeansGMM with EM algorithm

Example

A customer who buys both "sports equipment" and "books" might be 60% in Cluster A (fitness) and 40% in Cluster B (academics). Hard clustering forces a choice; probabilistic clustering captures both.

Probabilistic Membership
P(cₖ | xᵢ) = probability that point xᵢ belongs to cluster cₖ
Σₖ P(cₖ | xᵢ) = 1 (probabilities sum to 1 for each point)
The expected cluster for xᵢ is: argmax_k P(cₖ|xᵢ) — the cluster with highest probability
📝 RTU PYQs — Probabilistic Clustering
Q. Explain the working principle of Probabilistic clustering in Machine Learning with example.
May 2025 · Part B Q6 · 4 marks
Topic: Probabilistic Clustering · Unit 3
Probabilistic Clustering — Working Principle:

Probabilistic clustering models each cluster as a probability distribution. Each data point is assigned a probability of membership to each cluster rather than a hard assignment.

Key Idea: Assume data is generated from a mixture of k probability distributions (e.g., k Gaussian distributions). The goal is to find these distributions that best explain the data.

Algorithm (EM — Expectation Maximization):
Step 1 (E-step): For each point xᵢ and cluster k, compute:
  r(i,k) = P(cluster k | xᵢ) = πₖ·N(xᵢ|μₖ,Σₖ) / Σⱼ πⱼ·N(xᵢ|μⱼ,Σⱼ)
where r(i,k) is the "responsibility" of cluster k for point xᵢ.

Step 2 (M-step): Update parameters using responsibilities:
  μₖ = Σᵢ r(i,k)·xᵢ / Σᵢ r(i,k) (weighted mean)
  Σₖ = weighted covariance
  πₖ = Σᵢ r(i,k) / n (mixing proportion)

Example: 100 students — height and weight data. Some are tall-heavy (cluster 1 = athletes), some are short-light (cluster 2 = average). A medium-height, medium-weight student might be 60% likely in cluster 1 and 40% in cluster 2. Hard clustering forces a choice; probabilistic clustering keeps both probabilities.

Applications: Document clustering (a document about "sports medicine" partially belongs to both sports and medical clusters), customer segmentation, image segmentation.

Association Rule Mining

90% · Foundation for Apriori

What is Association Rule Mining?

Association Rule Mining finds interesting relationships (rules) between variables in large transactional databases. The classic example is market basket analysis — "customers who buy bread and butter also buy milk."

Rule format: X → Y (IF customer buys X, THEN they also buy Y)

Key Metrics — MUST MEMORIZE for Every Paper

Support (★ Most Important — Appears Every Year)
Support(X → Y) = Support(X ∪ Y) = count(X ∪ Y) / Total transactions
= Fraction of transactions that contain BOTH X and Y
Example: If {Bread, Butter} appears in 3 of 5 transactions → Support = 3/5 = 60%
Confidence
Confidence(X → Y) = Support(X ∪ Y) / Support(X)
= P(Y | X) = probability that Y is bought given X is bought
Example: Bread in 4 transactions, {Bread,Butter} in 3 → Conf(Bread→Butter) = 3/4 = 75%
Lift
Lift(X → Y) = Confidence(X → Y) / Support(Y)
Lift > 1: X and Y are positively correlated (buying X → more likely to buy Y)
Lift = 1: independent. Lift < 1: negatively correlated
⚡
Quick Calculation Trick

For RTU numericals: always convert min-sup to COUNT first!
min-sup = 60%, 5 transactions → min count = 5 × 0.6 = 3
min-sup = 22%, 9 transactions → min count = 9 × 0.22 = 1.98 → round up to 2
Now work with counts throughout — only convert back to % at the end.

Strong Association Rules

A rule X → Y is strong if BOTH conditions hold: Support ≥ min-sup AND Confidence ≥ min-conf.

Apriori Algorithm

🔥 92% — Numerical Every Year!

Core Idea — The Apriori Property

Apriori uses a clever mathematical property to efficiently find frequent itemsets without checking all 2^n subsets:

Apriori Property (Anti-Monotonicity)
If an itemset is INFREQUENT, then ALL its supersets are also INFREQUENT.
Equivalently: All subsets of a frequent itemset must also be frequent.
Example: If {A,B} is infrequent → {A,B,C} is definitely infrequent (no need to check!)

This property allows pruning — eliminating candidate itemsets early without scanning the database.

Apriori Algorithm Steps

1
Scan database: Find all frequent 1-itemsets (F₁). Any item with support ≥ min-sup.
2
Generate candidates C₂: Join F₁ with itself to form 2-itemset candidates.
3
Prune C₂: Remove any 2-itemset whose subsets (all 1-itemsets) are not in F₁.
4
Scan database: Count support for each candidate in C₂. Those with support ≥ min-sup form F₂.
5
Repeat: Generate Cₖ₊₁ from Fₖ, prune, scan for Fₖ₊₁ until no more frequent itemsets.
6
Generate rules: For each frequent itemset, generate all possible rules and keep those with confidence ≥ min-conf.
🧠
Prune Step Trick (2022 Part A Q3)

To prune Cₖ: for each candidate itemset of size k, check all its (k-1)-subsets. If ANY (k-1)-subset is NOT in F_{k-1}, remove the candidate. Example: Candidate {A,B,C} — check {A,B}, {A,C}, {B,C} all in F₂? If any is missing, prune {A,B,C}. Number of (k-1)-subsets of a k-itemset = C(k,k-1) = k subsets to check.

📝 RTU PYQs — Apriori
Q. Let C be a candidate item set in Cₖ generated by Apriori. How many length (K-1) subsets do we need to check in the prune step?
June 2022 · Part A Q3 · 2 marks
Topic: Apriori, Prune Step · Unit 3
For a candidate itemset C of size k, the number of (k-1)-length subsets = C(k, k-1) = k subsets.

Reasoning: Each (k-1)-subset is formed by removing one of the k items from C. Since there are k items to potentially remove, there are exactly k different (k-1)-subsets.

Example: C = {A,B,C} (k=3). We need to check 3 subsets: {A,B}, {A,C}, {B,C}. If ANY of these is not in F₂, then {A,B,C} is pruned from C₃.

FP-Growth Algorithm

🔥 90% — Part C Numerical

Why FP-Growth? — Apriori's Bottleneck

Apriori makes multiple database scans and generates huge candidate sets. For a 1000-item database, generating and checking all candidates is very slow. FP-Growth (Frequent Pattern Growth) solves this by compressing the database into an FP-Tree — a compact tree structure — and mining patterns directly from the tree without candidate generation.

AspectAprioriFP-Growth
Database scansk scans for k-itemsetsOnly 2 scans
Candidate generationYes (often many)No (no candidates)
MemoryHigh (large candidate sets)Compact tree
SpeedSlower for large dataMuch faster

FP-Growth Algorithm — Step by Step

1
Scan 1: Count item frequencies. Remove infrequent items (support < min-sup). Sort frequent items by descending frequency → get F-list.
2
Scan 2: Build FP-Tree. For each transaction: keep only frequent items, sort them in F-list order, insert into tree (sharing prefixes).
3
Mine FP-Tree: For each frequent item (from least to most frequent), find its conditional pattern base (paths in tree leading to that item).
4
Build conditional FP-Tree: Construct a new FP-Tree from the conditional pattern base.
5
Extract patterns: Frequent itemsets = item + frequent patterns in its conditional FP-tree.

Gaussian Mixture Model (GMM)

78% · Part B

What is GMM?

A Gaussian Mixture Model is a probabilistic model that assumes all data points are generated from a mixture of a finite number of Gaussian distributions with unknown parameters. It's the most common form of probabilistic clustering.

Intuition: Imagine height data for a group of adults. The data isn't one Gaussian — it's a mixture: one Gaussian for men (mean ~175cm) and one for women (mean ~162cm). GMM tries to discover both Gaussians from the mixed data.

GMM Probability Density
p(x) = Σₖ₌₁^K πₖ · N(x | μₖ, Σₖ)
πₖ = mixing coefficient (weight) for component k, Σπₖ = 1
N(x|μₖ,Σₖ) = Gaussian distribution with mean μₖ and covariance Σₖ
K = number of mixture components (like k in K-Means)

EM Algorithm for GMM

GMM parameters (πₖ, μₖ, Σₖ) are learned using the Expectation-Maximization (EM) algorithm:

E
E-step (Expectation): For each point xᵢ and component k, compute the responsibility: r(i,k) = πₖ·N(xᵢ|μₖ,Σₖ) / Σⱼ πⱼ·N(xᵢ|μⱼ,Σⱼ). This is the probability that component k "generated" point xᵢ.
M
M-step (Maximization): Update parameters using responsibilities: μₖ = Σᵢr(i,k)xᵢ / Σᵢr(i,k), Σₖ = weighted covariance, πₖ = (1/n)Σᵢr(i,k)
3
Repeat E and M steps until log-likelihood converges.

Connection: GMM → K-Means (2023 Part B Q3)

K-Means is a special case of GMM! When all Gaussian components have equal, spherical covariance (Σₖ = σ²I) and equal weights (πₖ = 1/k), and we take the hard assignment (highest responsibility wins), GMM reduces exactly to K-Means. RTU often asks about this relationship — always mention it in GMM answers!

📝 RTU PYQs — GMM
Q. What is the motivation behind the Gaussian Mixture Model? What is the relationship between k-means and PCA?
Aug 2023 · Part B Q3 · 4+4 marks
Topic: GMM, K-Means, PCA · Unit 3 + Unit 4
Motivation for GMM:
K-Means has limitations: it assumes spherical clusters, forces hard assignments, and cannot model uncertainty. Real-world data often forms elliptical, overlapping clusters. The motivation for GMM is:
1. Flexible cluster shapes: Gaussian distributions can model any elliptical shape via covariance matrices
2. Soft membership: Points between clusters get fractional membership (probability) rather than forced assignment
3. Probabilistic foundation: Enables Bayesian inference, likelihood estimation, and principled model selection
4. Generative model: Can generate new data samples from learned distributions
Example: Modelling heights in a mixed gender group — one Gaussian per gender, mixture models the full distribution.

Relationship between K-Means and PCA:
Both K-Means and PCA are dimensionality/complexity reduction techniques, but they work differently:
• K-Means finds cluster centroids — each centroid represents a cluster direction
• PCA finds principal components — directions of maximum variance
Formal relationship: K-Means with k=1 equivalent to finding the mean. With soft K-Means where membership weights are used in covariance estimation → approaches PCA. Specifically, if we relax K-Means to allow soft assignments and look at the first principal component of each cluster, it approximates what PCA does globally. Both try to find low-dimensional representations, just with different objectives.
Q. Suppose we clustered N data points using K-means and Gaussian mixtures (both k=5, same centers). Can three points assigned to different clusters in K-means be assigned to the same cluster in Gaussian mixture? Why or why not?
June 2022 · Part A Q6 · 2 marks
Topic: GMM vs K-Means · Unit 3
Yes, it is possible.

Reason: K-Means assigns each point to the nearest centroid (based on Euclidean distance alone — assumes spherical, equal-variance clusters). Gaussian mixture uses probability distributions that account for the shape and orientation of each cluster via covariance matrices.

If three points are equidistant from two centroids (borderline points), K-Means arbitrarily assigns each to its nearest centroid based on distance, potentially splitting them across clusters. GMM assigns them based on the full probability density of each Gaussian. If those three points happen to lie in a high-probability region of a single Gaussian (even if not the nearest centroid in Euclidean terms), GMM can assign all three to the same cluster.

Example: Points at (5,1), (5,2), (5,3) equidistant from centroids at (3,2) and (7,2). K-Means splits based on distance. But if the Gaussian at (3,2) has a large variance in the y-direction, all three points might have highest probability under that Gaussian → all in same GMM cluster.
