01

Course Objectives

Foundation · Conceptual

What This Course Aims to Teach

Machine Learning (6E7102 / 6CS4-02 / 6AID4-02) is a core sixth-semester paper for CS, IT, AID, and CAI branches under RTU Kota. It teaches algorithms that allow computers to learn patterns from data without being explicitly programmed for every rule.

By the end of this course, you will:

  1. Understand and implement supervised learning algorithms — regression, classification trees, SVM, and more (Unit 2).
  2. Apply unsupervised methods to discover hidden structure in unlabelled data using clustering and association rules (Unit 3).
  3. Use statistical tools — PCA, SVD, feature selection — to reduce dimensionality and evaluate models rigorously (Unit 4).
  4. Formulate reinforcement learning problems using MDPs and solve them with Q-learning, SARSA, and policy iteration (Unit 5–6).
  5. Choose the right algorithm for a given real-world problem and justify the choice using evaluation metrics.
02

What is Machine Learning?

Core Definition · RTU Favourite

The Classic Definition — Write This Verbatim in Exams

Arthur Samuel (1959) — the man who coined the term — defined Machine Learning as: "the field of study that gives computers the ability to learn without being explicitly programmed."

Tom Mitchell (1997) gave a more formal definition:

Mitchell's Formal Definition of ML A computer program is said to learn from experience E with respect to
some class of tasks T and performance measure P, if its performance
at tasks in T, as measured by P, improves with experience E.
Example: Spam filter. T = classify email as spam/not-spam. E = labelled emails. P = accuracy. The model improves as it sees more labelled examples.

How ML differs from traditional programming:

Traditional ProgrammingMachine Learning
Rules + Data → OutputData + Output → Rules (model)
Human encodes all logic explicitlyComputer discovers logic from examples
Brittle — breaks when rules changeAdapts as new data arrives
Example: if temperature > 37°C → feverExample: learns fever pattern from 10,000 patient records
⚡
Exam Trick — ML vs AI Distinction

RTU often asks: "Is ML just another name for AI? Comment." (2022 paper, Q1). The answer: No. AI is the broad field of making machines intelligent. ML is a subset of AI that specifically uses data-driven learning. All ML is AI, but not all AI is ML (rule-based expert systems are AI but not ML). Draw a Venn diagram: AI ⊃ ML ⊃ Deep Learning. This earns full 2 marks.

03

Types of Machine Learning

Core Framework · Maps to All Units

The Four Learning Paradigms

Every ML algorithm falls into one (or more) of these four categories. Understanding these gives you a mental map of the entire course:

Supervised Learning

Learn a mapping from inputs X to outputs Y using labelled training data (X, Y pairs). You know the correct answer for each training example.

→ Unit 2 · Regression, Classification
Unsupervised Learning

Find hidden structure in unlabelled data. No correct answers given — the algorithm discovers groups or patterns on its own.

→ Unit 3 · Clustering, Association Rules
Semi-supervised Learning

A mix — small amount of labelled data + large amount of unlabelled data. Labels are expensive; semi-supervised exploits unlabelled data.

→ Unit 5 · Text Classification, Medical Imaging
Reinforcement Learning

An agent learns to act in an environment by trial and error — receiving rewards for good actions and penalties for bad ones. No labelled dataset.

→ Unit 5–6 · MDP, Q-Learning, SARSA

Supervised Sub-types — Classification vs Regression

TypeOutputExampleAlgorithms
ClassificationDiscrete categorySpam / Not-Spam; Disease: Yes/NoDecision Tree, Naive Bayes, KNN, SVM, Logistic Regression, Random Forest
RegressionContinuous valueHouse price prediction; Sales forecastLinear Regression, Ridge, Lasso, SVR
Key Difference — Classification vs Regression (2023, 2022 PYQ) Classification: f: X → {C₁, C₂, …, Cₖ} Output is a class label (discrete)
Regression: f: X → ℝ Output is a real number (continuous)
Similarity: Both are supervised — both need labelled training data. The loss function differs: Classification uses cross-entropy; Regression uses MSE.
04

Key ML Terminology

Vocabulary · Part A Ready

Terms You Must Know for Part A (2-mark answers)

TermDefinition (≤ 25 words)
Feature (Attribute)An individual measurable property or input variable used to describe a data point. E.g., age, salary.
Label (Target)The output variable the model is trying to predict. Known in training, unknown in production.
Training DataThe labelled dataset used to fit the model parameters during learning.
OverfittingModel learns training data too well — memorises noise. High training accuracy, low test accuracy.
UnderfittingModel is too simple to capture the pattern. Low accuracy on both training and test data.
BiasError from wrong assumptions — model is too rigid. High bias = underfitting.
VarianceError from sensitivity to small training data fluctuations. High variance = overfitting.
Cross-validationTechnique to estimate generalisation by training/testing on different data splits (e.g., k-fold).
Confusion MatrixTable showing TP, TN, FP, FN counts for a classifier. Derives precision, recall, F1-score.
False Negative (FN)Positive case predicted as negative. E.g., cancer patient classified as healthy. Type II error.
Confusion Matrix — Must Memorise for Every Paper Precision = TP / (TP + FP) ← "Of all predicted positive, how many actually are?"
Recall = TP / (TP + FN) ← "Of all actual positive, how many did we catch?"
F1-Score = 2 × (P × R) / (P + R) ← Harmonic mean — balances both
Accuracy = (TP + TN) / Total
False Negative is more dangerous in medical diagnosis (missed cancer). False Positive is more dangerous in spam (important email deleted).
✅
Overfitting vs Underfitting — Draw This in Every Exam

Draw three curves: training error and test error vs. model complexity. Underfitting = both errors high (left). Good fit = both errors low (middle). Overfitting = training error low, test error high (right). The gap between training and test error IS the variance. This diagram earns you 3 marks minimum in any related question.

10-Mark Answer Template (Part C)

  1. Definition/Introduction — 2–3 lines. Use Mitchell's definition if ML-general.
  2. Key Formula / Mathematical Model — Box it. Examiner scans for this first.
  3. Algorithm Steps — Numbered list. "Step 1: Initialise… Step 2: Compute…"

  5. Worked Example — Small numerical demonstrates full understanding.
  6. Advantages / Disadvantages / Applications — 2 points each fills remaining space.
⚡
The Diagram Rule — Never Skip It

RTU ML examiners explicitly award marks for diagrams (the instruction sheet says "schematic diagrams must be shown wherever necessary"). For Decision Tree: draw the tree structure with if/else branches. For Neural Network: draw nodes, layers, arrows, activation function. For K-Means: draw centroid movement iterations. For MDP: draw the state-action-reward-state diagram. A well-drawn labelled diagram can earn you 3–4 marks even when your written text is thin.

Solved PYQs — Unit 1 Topics

Part A · 2 Marks Each
📋 2022 PAPER · PART A Q1 Topic: ML vs AI · Unit 1 2 Marks
"Is Machine Learning just another name for Artificial Intelligence?" Comment.
No. AI is the broad discipline of creating intelligent machines. ML is a subset of AI that specifically focuses on algorithms which learn patterns from data without explicit programming. All ML systems are AI, but not all AI systems use ML — rule-based expert systems (e.g., chess engines before deep learning) are AI but not ML. The hierarchy is: AI ⊃ ML ⊃ Deep Learning.
📋 2023 PAPER · PART A Q1 Topic: Classification vs Regression · Unit 1 2 Marks
What is the difference between classification and regression?
Classification: Predicts a discrete class label. Output ∈ {C₁, C₂, …, Cₖ}. E.g., email → spam/not-spam. Uses algorithms: Decision Tree, Naive Bayes, SVM.

Regression: Predicts a continuous numerical value. Output ∈ ℝ. E.g., predicting house price in rupees. Uses: Linear Regression, SVR.

Similarity: Both are supervised — both require labelled training data. Both build a model f(X) from examples.
📋 2023 PAPER · PART A Q3 Topic: Overfitting & Underfitting · Unit 1 2 Marks
Explain overfitting and underfitting.
Overfitting: The model learns the training data too well — including its noise and random fluctuations — and fails to generalise to new data. High training accuracy, low test accuracy. Caused by: too complex model, too little data. Solution: regularisation, more data, pruning (for decision trees).

Underfitting: The model is too simple to capture the true patterns. Both training and test accuracy are low. Caused by: too few features, too simple a model. Solution: add features, use a more complex algorithm.

Bias-Variance Tradeoff: Overfitting = High Variance. Underfitting = High Bias. Ideal model minimises both.
📋 2025 PAPER · PART A Q3 Topic: Confusion Matrix · Unit 1 2 Marks
What is Confusion Matrix?
A Confusion Matrix is an N×N table (N = number of classes) that summarises classifier performance by showing the counts of correct and incorrect predictions for each class.

For binary classification (2×2 matrix):
  • TP (True Positive): predicted positive, actually positive ✓
  • TN (True Negative): predicted negative, actually negative ✓
  • FP (False Positive): predicted positive, actually negative ✗ — Type I error
  • FN (False Negative): predicted negative, actually positive ✗ — Type II error

Derived metrics: Accuracy = (TP+TN)/Total, Precision = TP/(TP+FP), Recall = TP/(TP+FN), F1 = 2PR/(P+R).
📋 2024 PAPER · PART A Q5 Topic: False Negative · Unit 1 2 Marks
What do you understand by false negative in confusion matrix?
False Negative (FN) — also called a Type II Error — occurs when the model predicts the negative class but the actual label is positive.

Example: A patient has cancer (positive), but the classifier predicts "no cancer" (negative). This is a False Negative — a missed detection — which is medically dangerous.

FN is captured by Recall: Recall = TP/(TP+FN). High FN → Low Recall. In critical applications (medical, fraud), we want to minimise FN even at the cost of more FP.
