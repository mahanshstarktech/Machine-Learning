Recommender Systems

80% · Part A/B

What is a Recommender System?

A recommender system is a type of information filtering system that predicts and suggests items a user might like, based on their past behaviour, preferences, or characteristics of items.

Real-world examples: Netflix suggesting movies, Amazon recommending products, Spotify creating playlists, YouTube auto-play, Google News personalization.

The core problem: Users haven't seen all items. The system must predict rating/preference for unseen items and recommend the highest predicted ones.

Types of Recommender Systems

Similarity Measures Used in Recommender Systems (2023 Part A Q10)

Cosine Similarity (Most Common)
cos(u,v) = (u · v) / (||u|| × ||v||) = Σuᵢvᵢ / √(Σuᵢ²) × √(Σvᵢ²)
Range: [-1,1]. 1 = identical, 0 = orthogonal, -1 = opposite
Pearson Correlation
r(u,v) = Σ(rᵤᵢ - r̄ᵤ)(rᵥᵢ - r̄ᵥ) / √[Σ(rᵤᵢ - r̄ᵤ)² × Σ(rᵥᵢ - r̄ᵥ)²]
Accounts for different rating scales (user bias). Better for explicit ratings.

Aug 2023 · Part A Q10 · 2 marks
Topic: Recommender Systems · Unit 6
Techniques for finding similarity in recommender systems:

1. Cosine Similarity: Measures the cosine of the angle between two user/item vectors. cos(u,v) = (u·v)/(||u||×||v||). Range [0,1]. Ignores magnitude — focuses on direction (preferences pattern).

2. Pearson Correlation: Measures linear correlation between ratings, accounting for user's average rating bias. Range [-1,1]. More robust than cosine for explicit rating data.

3. Euclidean Distance: d(u,v) = √Σ(uᵢ-vᵢ)². Smaller distance = more similar. Simple but sensitive to scale.

4. Jaccard Similarity: For binary data (liked/not-liked). |A∩B|/|A∪B|. Used in implicit feedback systems.

Collaborative Filtering

85% · Part B

Core Idea

Collaborative Filtering (CF) recommends items based on the collective behaviour of many users. It assumes users with similar past behaviour will like similar things in the future — "wisdom of the crowd."

It doesn't need item content information — only needs past user-item interactions (ratings, clicks, purchases).

Two Types of CF

TypeHow it worksStepsBest for
User-based CFFind users similar to active user → Recommend what similar users liked1. Build user-item matrix. 2. Compute user-user similarity. 3. Find k-nearest similar users. 4. Predict ratings. 5. Recommend top items.Small user base, changing items
Item-based CFFind items similar to what user already liked → Recommend similar items1. Build item-item similarity matrix. 2. For each item user rated, find similar items. 3. Predict rating for unseen items. 4. Recommend top items.Large user base, stable items

Rating Prediction Formula

User-based CF Prediction
r̂(u,i) = r̄ᵤ + Σₙ sim(u,n)×(r(n,i) - r̄ₙ) / Σₙ |sim(u,n)|
r̂(u,i) = predicted rating of user u for item i
r̄ᵤ = mean rating of user u (accounts for user bias)
sim(u,n) = similarity between users u and n
Sum over N most similar users who rated item i

Advantages & Disadvantages

AdvantagesDisadvantages
No need to understand item contentCold start problem — can't recommend to new users/items
Can discover serendipitous recommendationsScalability — computing similarity for millions of users is expensive
Domain independent (works for any item type)Sparsity — most users rate very few items
Improves with more usersPopularity bias — popular items always recommended

Q. What are Recommender Systems? What is Collaborative filtering and Content Based filtering?
Aug 2023 · Part B Q2 · 4+4 marks
Topic: Collaborative & Content-Based Filtering · Unit 6
Recommender Systems:
Systems that filter information to predict user preferences and suggest relevant items. Used in e-commerce (Amazon), streaming (Netflix, Spotify), and social media.

Collaborative Filtering:
Recommends items based on the preferences of similar users (or based on item similarity). Does NOT need item content — relies purely on user-item interaction data.

Two approaches:
1. User-based: "Users like you also liked..." — find users with similar ratings history, recommend what they liked.
2. Item-based: "You liked X, others who liked X also liked Y..." — find items similar to user's liked items.

Example: You and User B both rated Movie A=5, Movie B=4. User B also rated Movie C=5. CF recommends Movie C to you.

Limitations: Cold start (new users/items), sparsity, scalability.

Content-Based Filtering:
Recommends items similar to what a user has liked before, based on the FEATURES/CONTENT of items. Builds a user profile from features of liked items.

Steps: 1. Extract item features (genre, director, keywords). 2. Build user profile (weighted average of features of liked items). 3. Match user profile against all items using cosine similarity. 4. Recommend items with highest similarity score.

Example: You liked "Action movies with Tom Cruise" → system recommends other Action movies with Tom Cruise (based on item features).

Limitations: Over-specialisation (can't recommend new types), requires good feature extraction, can't find serendipitous items.

2024 · Part A Q3 · 2 marks
Topic: Content-Based Filtering · Unit 6
Advantages of Content-Based Filtering:
1. No cold-start for items: Can recommend new items immediately as long as their features are known — doesn't need rating history for items.
2. No sparsity problem: Doesn't depend on other users' ratings — works even with few users.
3. Transparent: Can explain recommendations — "We recommended this because you liked X which has similar genre/features."
4. User independence: Recommendations are personalised to the individual — not influenced by what other users do.
5. Niche items: Can recommend unpopular but relevant items that CF might miss.

Content-Based Filtering

75% · Part A/B

How Content-Based Filtering Works

Content-Based Filtering recommends items by comparing the FEATURES of items with a user's preference profile. It answers: "Given what you've liked before (in terms of content characteristics), what else would you like?"

1
Feature Extraction: Represent each item as a feature vector. For movies: [genre, director, cast, year, keywords]. For news: TF-IDF of article text.
2
User Profile Construction: Create a user profile = weighted average of feature vectors of items the user has rated. Higher-rated items contribute more.
3
Similarity Matching: For each unseen item, compute similarity between item's feature vector and user's profile vector (cosine similarity).
4
Recommendation: Rank items by similarity score. Recommend top-N items.

Artificial Neural Networks (ANN)

🔥 90% · Part B/C

Biological Inspiration

The human brain has ~86 billion neurons, each connected to ~10,000 others via synapses. Artificial Neural Networks are computational models loosely inspired by this structure. They learn by adjusting connection weights based on examples — just as biological synapses strengthen with repeated activation.

Biological vs Artificial neuron:

Biological NeuronArtificial Neuron
Dendrites (receive signals)Inputs (x₁, x₂, ..., xₙ)
Synapse strengthWeights (w₁, w₂, ..., wₙ)
Cell body (sum + threshold)Summation + Activation function
Axon (output signal)Output y

Mathematical Model of a Single Neuron

Single Neuron Computation
Net Input: h = w₀ + w₁x₁ + w₂x₂ + ... + wₙxₙ = bias + Σ wᵢxᵢ
Output: y = f(h) = activation_function(h)
w₀ = bias weight (threshold), wᵢ = input weights, xᵢ = inputs

Common Activation Functions (Must Know)

FunctionFormulaRangeUse
Step (Threshold)f(h) = 1 if h>0, else 0{0,1}Perceptron
Sigmoidf(h) = 1/(1+e⁻ʰ)(0,1)Binary classification output
Tanhf(h) = (eʰ-e⁻ʰ)/(eʰ+e⁻ʰ)(-1,1)Hidden layers
ReLUf(h) = max(0,h)[0,∞)Deep learning hidden layers
Softmaxf(hᵢ) = eʰⁱ / Σeʰʲ(0,1), Σ=1Multi-class output

Topic: ANN Structure · Unit 6
An ANN consists of interconnected layers of artificial neurons (nodes):

1. Input Layer: Receives raw input features x₁, x₂, ..., xₙ. No computation — just distributes inputs to next layer.

2. Hidden Layer(s): One or more intermediate layers. Each neuron computes: h = f(Σwᵢxᵢ + bias). The function f is a non-linear activation function (sigmoid, tanh, ReLU).

3. Output Layer: Produces final predictions ŷ. Number of neurons = number of output classes (for classification) or 1 (for regression).

Connections: Fully connected between layers. Each connection has a learnable weight w. Learning = adjusting these weights via backpropagation to minimize loss.

Perceptron

88% · Numerical

What is a Perceptron?

The Perceptron is the simplest form of ANN — a single layer of neurons that can learn to classify linearly separable data. Invented by Frank Rosenblatt in 1957.

Single-layer perceptron can only solve linearly separable problems (AND, OR gates — but NOT XOR). For XOR, you need a multilayer perceptron.

Perceptron Computation
Net input: h = w₀·(bias) + w₁x₁ + w₂x₂ + ... + wₙxₙ
Output: y = 1 if h > 0, else 0 (threshold activation)

Perceptron Learning Rule

Weight Update Rule (★ Exam Must-Memorize)
Δwᵢ = η × (target − output) × xᵢ
wᵢ(new) = wᵢ(old) + Δwᵢ
η = learning rate. target t ∈ {0,1}. output y ∈ {0,1}.
If output = target (correct): Δwᵢ = 0 (no change)
If output = 0, target = 1 (missed): Δwᵢ = +η × xᵢ (increase weights for active inputs)
If output = 1, target = 0 (false alarm): Δwᵢ = -η × xᵢ (decrease weights)

⭐⭐⭐ RTU Numerical — Perceptron Training (2023 Part C Q5)

📌 Perceptron on Truth Table — Full Solution

Given: Initial weights w₀=0.03, w₁=-0.04, w₂=0.04. Learning rate η=0.4. Bias=-1. Activation: g(h)=1 if h>0, g(h)=0 if h≤0.

Training data (NOR-like pattern):

X    Y    Target    (x₁,x₂ = X,Y)
1    1    0
1    0    0
0    1    1
0    0    1

Iteration 1 — Sample (1,1), Target=0

h = w₀×(-1) + w₁×1 + w₂×1
  = 0.03×(-1) + (-0.04)×1 + 0.04×1
  = -0.03 - 0.04 + 0.04 = -0.03

g(-0.03) = 0 (since -0.03 ≤ 0)
output = 0, target = 0 → CORRECT, no update

Sample (1,0), Target=0

h = 0.03×(-1) + (-0.04)×1 + 0.04×0
  = -0.03 - 0.04 + 0 = -0.07
g(-0.07) = 0 = target → CORRECT, no update

Sample (0,1), Target=1

h = 0.03×(-1) + (-0.04)×0 + 0.04×1
  = -0.03 + 0 + 0.04 = 0.01
g(0.01) = 1 = target → CORRECT, no update

Sample (0,0), Target=1

h = 0.03×(-1) + (-0.04)×0 + 0.04×0
  = -0.03 + 0 + 0 = -0.03
g(-0.03) = 0, but target = 1 → ERROR!

Update: Δwᵢ = η × (target - output) × xᵢ
Δw₀ = 0.4 × (1-0) × (-1) = -0.4    w₀_new = 0.03 + (-0.4) = -0.37
Δw₁ = 0.4 × (1-0) × 0 = 0          w₁_new = -0.04
Δw₂ = 0.4 × (1-0) × 0 = 0          w₂_new = 0.04

Updated weights: w₀=-0.37, w₁=-0.04, w₂=0.04

Continue iterations until all 4 samples produce correct output in one pass (convergence). RTU expects 2-3 iterations shown.

✅
Validation: Check All Outputs

After each complete pass through all samples, verify all 4 outputs = targets. If YES → converged. If NO → do another iteration. Perceptron convergence theorem: if data is linearly separable, the algorithm WILL converge in finite iterations.

Multilayer Network

88% · Short Notes

Why Multilayer? — XOR Problem

A single-layer perceptron can ONLY learn linearly separable patterns. XOR is NOT linearly separable (you cannot draw a single straight line to separate 0s and 1s). Adding hidden layers solves this.

A Multilayer Perceptron (MLP) has one or more hidden layers between input and output. Each hidden layer transforms the data into a new representation — ultimately making even non-linear problems linearly separable in the final transformed space.

Fig 6.1 — XOR requires hidden layer; single layer fails

Key Properties of Multilayer Networks

  • Universal Function Approximation: A network with one hidden layer and sufficient neurons can approximate any continuous function (Universal Approximation Theorem).
  • Non-linear transformations: Each hidden layer applies a non-linear activation, allowing the network to learn complex patterns.
  • Representation learning: Each layer learns increasingly abstract features — pixels → edges → shapes → objects (in image recognition).

Forward Pass Computation

Forward Propagation (Layer by Layer)
Layer 1 (hidden): h = f(W¹ · x + b¹)
Layer 2 (output): ŷ = g(W² · h + b²)
W¹,W² = weight matrices, b¹,b² = bias vectors, f,g = activation functions
Matrix multiply → add bias → apply activation → next layer

2-Layer vs 3-Layer Neural Network (2024 Part A Q2)

Aspect2-Layer (1 hidden)3-Layer (2 hidden)
ArchitectureInput → Hidden → OutputInput → Hidden1 → Hidden2 → Output
ComplexityCan solve most non-linear problemsCan solve more complex patterns
Training timeFasterSlower
Vanishing gradient riskLowHigher (more layers)
Feature abstraction1 level of abstraction2 levels of abstraction

Q. Suppose you had a neural network with linear activation functions. Show there is a network with no hidden units that computes the same function.
June 2022 · Part C Q5b · 8 marks
Topic: Linear Activation, Multilayer Networks · Unit 6
Key insight: If all activation functions are linear, f(x) = cx, then the entire multilayer network collapses to a single linear transformation, regardless of depth.

Proof for 1 hidden layer:
Let network: Input x → Hidden h → Output y
h = c·(W¹x + b¹) = cW¹x + cb¹
y = c·(W²h + b²) = c·W²·(cW¹x + cb¹) + cb²
y = c²W²W¹x + c²W²b¹ + cb²
y = W_effective·x + b_effective
where W_effective = c²W²W¹ (a single weight matrix) and b_effective is a bias vector.

This is exactly a single-layer (no hidden layer) network with weight W_effective! So the hidden layer adds NO expressive power.

Conclusion: With linear activation functions, no matter how many hidden layers, the network is equivalent to a single linear layer (single matrix multiplication). This is why NON-LINEAR activations (sigmoid, ReLU, tanh) are essential — they prevent this collapse and give deep networks their expressive power.

Backpropagation Algorithm

🔥 92% — Most Asked!

The Big Idea — Learning from Mistakes

Backpropagation is the algorithm used to train multilayer neural networks. It works by computing the gradient of the loss function with respect to each weight using the chain rule of calculus, then updating weights in the direction that reduces the loss.

Two passes for each training example:

  • Forward pass: Compute output and loss (error) — data flows forward.
  • Backward pass: Compute gradients and update weights — error signals flow backward.

Backpropagation — Step by Step

1
Initialize weights randomly (small values near 0).
2
Forward Pass: For input x, compute activations layer by layer: hˡ = f(Wˡ·hˡ⁻¹ + bˡ). Compute output ŷ = hᴸ.
3
Compute Loss: E = ½(t - ŷ)² for MSE, or cross-entropy for classification. This is the error to minimize.
4
Backward Pass — Output Layer: Compute output delta: δᴸ = (t - ŷ) × f'(hᴸ)
5
Backward Pass — Hidden Layers: Propagate delta backwards: δˡ = (Wˡ⁺¹ᵀ · δˡ⁺¹) × f'(hˡ)
6
Update Weights: ΔWˡ = η × δˡ × (hˡ⁻¹)ᵀ. New weight: Wˡ ← Wˡ + ΔWˡ
7
Repeat Steps 2–6 for all training examples and multiple epochs until loss converges.
Key Backprop Formulas (★★★ Memorize)
Loss: E = ½ Σ(tₖ - yₖ)² (sum over output neurons)
Output delta: δₖ = (tₖ - yₖ) · f'(netₖ) where f'(x) = f(x)·(1-f(x)) for sigmoid
Hidden delta: δⱼ = f'(netⱼ) · Σₖ δₖ · wⱼₖ
Weight update: Δwᵢⱼ = η · δⱼ · yᵢ (from layer i to layer j)
η = learning rate. f'(x) = sigmoid derivative. Chain rule applied backwards.

Sigmoid Derivative (Important for Numericals)

sigmoid: σ(x) = 1/(1 + e⁻ˣ)
derivative: σ'(x) = σ(x) × (1 - σ(x))
This property makes backprop efficient — the derivative is computable from the output itself!

Introduction to Deep Learning

88% · Part B

What is Deep Learning?

Deep Learning is a subfield of machine learning that uses neural networks with many hidden layers (hence "deep") to automatically learn hierarchical feature representations from raw data. The depth enables learning at multiple levels of abstraction.

AspectTraditional MLDeep Learning
Feature engineeringManual (domain expert designs features)Automatic (learns features from raw data)
Data requirementWorks with small dataNeeds large data (millions of examples)
Compute requirementCPU sufficientGPU/TPU often needed
InterpretabilityMostly interpretable"Black box" — hard to interpret
Performance on unstructured dataModerateState-of-the-art (images, text, speech)

Key Deep Learning Architectures

1. Convolutional Neural Networks (CNN)

Specialized for image data. Uses convolutional layers to detect local patterns (edges, textures, shapes). Pooling layers reduce spatial dimensions. Applications: image classification, object detection, face recognition.

CNN Layers: Input → [Conv → ReLU → Pool]ⁿ → Flatten → FC → Output

2. Recurrent Neural Networks (RNN)

Specialized for sequential data. Has loops — output from previous step feeds back as input. Maintains "memory" of past steps. Applications: text, speech, time series. Variant: LSTM (Long Short-Term Memory) handles long dependencies.

3. Autoencoders

Encoder compresses input to low-dimensional representation (latent space). Decoder reconstructs from latent space. Applications: dimensionality reduction, anomaly detection, generative models.

4. Transformers (Most Recent)

Uses attention mechanism — processes all input simultaneously, not sequentially. Foundation of GPT, BERT, ChatGPT. State-of-the-art for NLP.

Key Deep Learning Concepts

Vanishing/Exploding Gradient Problem

In deep networks, gradients can become extremely small (vanishing) or large (exploding) during backpropagation, making training impossible.

Solutions: ReLU activation (doesn't saturate), Batch Normalization, Residual connections (skip connections in ResNet), Gradient clipping for exploding.

Dropout — Regularisation for Deep Networks

During training, randomly "drop" (zero out) some neurons with probability p. Forces network to learn redundant representations. Prevents overfitting. At test time, use all neurons with scaled weights.

Transfer Learning

Use a pre-trained deep network (trained on ImageNet) and fine-tune on your specific task. Dramatically reduces data and compute requirements. Foundation of modern applied deep learning.

Q. Explain why precision or recall alone is not a sufficient measure. Discuss AUC-ROC curve in classification problems.
Aug 2023 · Part C Q2 · 15 marks (overlaps Unit 4)
Topic: Model Evaluation, AUC-ROC · Unit 4 + Unit 6
Why Precision and Recall alone are insufficient:
• Precision = TP/(TP+FP): "Of all predicted positives, how many are truly positive?" High precision alone is achievable by making very few positive predictions.
• Recall = TP/(TP+FN): "Of all actual positives, how many did we catch?" High recall alone is achievable by predicting everything as positive.

These metrics trade off against each other: You can always increase recall by predicting more positives (but precision drops), or increase precision by predicting fewer positives (but recall drops). Neither metric alone captures both aspects. F1-score (harmonic mean) balances them, but it uses a fixed threshold.

AUC-ROC Curve:
ROC = Receiver Operating Characteristic. Plots:
• X-axis: False Positive Rate (FPR) = FP/(FP+TN) — at each threshold
• Y-axis: True Positive Rate (TPR) = Recall = TP/(TP+FN) — at each threshold

By varying the classification threshold from 0 to 1, we get a curve. AUC = Area Under this Curve.

AUC interpretation:
• AUC = 1.0: Perfect classifier
• AUC = 0.5: Random classifier (diagonal line)
• AUC > 0.7: Good model; AUC > 0.9: Excellent

Advantage of AUC: Threshold-independent — evaluates classifier performance across ALL possible thresholds, not just one. Good for imbalanced datasets where accuracy is misleading. [Draw ROC curves with diagonal reference line]
