Introduction to Supervised Learning

90% · Foundation

What is Supervised Learning?

In supervised learning, the algorithm is trained on a labelled dataset — a set of input-output pairs (X, Y). The goal is to learn a mapping function f such that f(X) ≈ Y. The word "supervised" means a "teacher" (the labels) guides the learning.

Supervised Learning — Mathematical Formulation
Given: Training set D = {(x₁,y₁), (x₂,y₂), ..., (xₙ,yₙ)}
Find: f: X → Y such that loss L(f(xᵢ), yᵢ) is minimized
x = feature vector (input), y = label (output), n = number of training examples

Two Types of Supervised Learning

Memory Trick — Extraction vs Selection

EXTRACT = create NEW (like extracting juice — you get something new, not the original). SELECT = CHOOSE from existing (like selecting players from a squad — original players, just fewer). In extraction, old features are gone; in selection, rejected features are simply not used.

📝 PYQs — Introduction / Feature Concepts
Q. Differentiate between feature extraction and selection.
2024 Part A Q4 · 2022 Part A Q5 · 2 marks each
Topic: Feature Extraction vs Selection · Unit 2/4
Feature Extraction: Transforms the original feature space into a new, lower-dimensional feature space using mathematical operations. New features are combinations of original ones. E.g., PCA creates orthogonal principal components from original features. The original features are no longer directly used.

Feature Selection: Selects a meaningful subset of the original features by eliminating redundant or irrelevant ones. The original feature values are retained, just fewer of them. E.g., selecting only "Age" and "Blood Pressure" from 50 medical features for diabetes prediction.

Key Difference: Extraction creates new features; Selection chooses existing ones.
Similarity: Both reduce dimensionality, reduce overfitting, and improve model performance.

Linear Regression

🔥 85% — Numerical Always Asked

What is Linear Regression?

Linear Regression is the simplest supervised learning algorithm for predicting a continuous output. It assumes a linear relationship between input variables (features) and the output.

Intuition: Imagine plotting midterm scores (x) vs final scores (y) on a graph. Linear regression draws the best-fit straight line through these points. Given a new midterm score, you read off the predicted final score from this line.

Linear Regression Model (Simple — 1 Feature)
ŷ = mx + c   (or)   ŷ = β₀ + β₁x
where: ŷ = predicted output, x = input feature
β₀ = intercept (c), β₁ = slope (m)
The line minimizes the sum of squared errors between predicted ŷ and actual y.
Least Squares Formulas — MUST MEMORIZE (★★★)
β₁ = [n·Σ(xᵢyᵢ) − Σxᵢ·Σyᵢ] / [n·Σ(xᵢ²) − (Σxᵢ)²]
β₀ = ȳ − β₁·x̄
where: x̄ = mean of x, ȳ = mean of y, n = number of data points
This is the "normal equation" solution — exact, closed-form. Always write the formula first, then substitute values in a table.

Loss Function — Why "Least Squares"?

We find β₀ and β₁ that minimize the Mean Squared Error (MSE):

Loss Function — Mean Squared Error
MSE = (1/n) · Σᵢ (yᵢ − ŷᵢ)² = (1/n) · Σᵢ (yᵢ − β₀ − β₁xᵢ)²
Squaring: (1) makes all errors positive, (2) penalizes large errors more. Minimizing this analytically gives the least squares formulas above.

Multiple Linear Regression (More Features)

ŷ = β₀ + β₁x₁ + β₂x₂ + ... + βₖxₖ
In matrix form: ŷ = Xβ, where β = (XᵀX)⁻¹Xᵀy (Normal Equation)

Assumptions of Linear Regression (Theory Filler)

  • Linearity: Relationship between X and Y is linear
  • Independence: Observations are independent of each other
  • Homoscedasticity: Constant variance in errors at every X value
  • Normality: Errors are normally distributed
  • No multicollinearity: Features are not highly correlated with each other

Naive Bayes Classifier

75% · Part A/B

What is Naive Bayes?

Naive Bayes is a probabilistic classifier based on Bayes' Theorem. It is called "naive" because it makes a simplifying assumption: all features are conditionally independent given the class label — which is rarely true in reality but works remarkably well in practice.

Intuition: Spam filter. Given an email with words "lottery", "win", "free" — what's the probability it's spam? Naive Bayes multiplies individual word probabilities together, assuming each word's presence is independent of others.

Bayes' Theorem — Foundation
P(C|X) = P(X|C) · P(C) / P(X)
where: C = class, X = feature vector = (x₁, x₂, ..., xₙ)
P(C|X) = Posterior probability (what we want)
P(X|C) = Likelihood (probability of features given class)
P(C) = Prior probability (class frequency in training data)
P(X) = Evidence (constant — same for all classes, ignored)
Naive Bayes Classification Rule
Predicted class = argmax_C [P(C) · Π P(xᵢ|C)]
= argmax_C [P(C) · P(x₁|C) · P(x₂|C) · ... · P(xₙ|C)]
The "naive" assumption: P(X|C) = P(x₁|C)·P(x₂|C)·...·P(xₙ|C) — features are independent.

Why is it Called "Naive"? (2022 Part B Q2a)

The classifier is called "naive" because it makes the naive independence assumption: it assumes all features (x₁, x₂, ..., xₙ) are mutually independent given the class label C.

This is almost never true in real data. For example, in a spam classifier, the words "lottery" and "win" are NOT independent — they frequently appear together. But despite this unrealistic assumption, Naive Bayes performs surprisingly well in text classification, medical diagnosis, etc.

Advantages of Naive Bayes (2024 Part B Q6)

  • Fast: Training = counting frequencies. Prediction = few multiplications. O(n) complexity.
  • Works well with small data: Needs fewer training examples than discriminative models.
  • Handles high dimensions well: 10,000 features (e.g., words) is fine — each is estimated independently.
  • Multi-class naturally: No modification needed for more than 2 classes.
  • Online learning: New data can be incorporated incrementally.

Steps of Naive Bayes Algorithm

1
Calculate Prior: P(C) = count of class C / total examples. (E.g., P(Spam) = 60/100 = 0.6)
2
Calculate Likelihood: For each feature xᵢ and class C, compute P(xᵢ|C) from training data.
3
Apply Bayes: For new data point X, compute P(C|X) ∝ P(C)·Π P(xᵢ|C) for each class.
4
Classify: Predict the class with highest posterior probability.
Laplace Smoothing (Zero-Frequency Problem)
P(xᵢ = v|C) = (count(xᵢ = v, C) + 1) / (count(C) + |V|)
If a word never appeared in training for a class, P(word|class)=0 → kills entire product. Laplace smoothing adds 1 to all counts. |V| = vocabulary size.
🔗
Connection to Decision Tree

Both Naive Bayes and Decision Trees are classifiers. Decision Tree uses entropy/Gini to split features — a deterministic rule. Naive Bayes uses probability — a soft probabilistic rule. Decision Tree works better when features interact (non-independence). Naive Bayes works better with high-dimensional sparse data (text). Together, they're two of the most widely used classifiers in RTU exam answers.

June 2022 · Part B Q2a · 4 marks
Topic: Naive Bayes "naive" assumption · Unit 2
It is called "naive" due to the naive independence assumption: the classifier assumes all features are conditionally independent given the class label.

Formally: P(X|C) = P(x₁|C) · P(x₂|C) · ... · P(xₙ|C)

This simplification allows computing the joint probability of all features as a simple product of individual feature probabilities. However, in practice, features are rarely independent. 2. The independence assumption prevents overfitting.

2024 · Part B Q6 · 4 marks
Topic: Naive Bayes Advantages + Algorithm · Unit 2
Advantages:
1. Extremely fast — training is just counting, prediction is multiplications
2. Works with small datasets — needs few training examples
3. Handles high-dimensional data well (text with thousands of words)
4. Naturally multi-class without modification
5. Not sensitive to irrelevant features (they contribute equally small probabilities)

Major Steps:

Step 2: Prior probability: P(C) = (# instances of class C) / (total instances) for each class C.

Step 5: Predict: Assign class with maximum score. Class = argmax_C Score(C).

Decision Tree (ID3 / CART)

🔥 95% — Most Important in Unit 2

What is a Decision Tree?

A Decision Tree is a flowchart-like tree structure where each internal node represents a "test" on a feature, each branch represents an outcome, and each leaf node represents a class label (for classification) or value (for regression).

Intuition: A game of "20 Questions." You ask yes/no questions to narrow down possibilities. Each question splits the space. The most informative question first gives you the deepest split. That's exactly what a decision tree does — it finds the most "informative" feature at each step.

Fig 4.1 — Decision Tree for "Play Tennis" problem. Each internal node = feature test. Leaves = class labels.

Key Measure 1 — Entropy (ID3 Algorithm)

Entropy measures the impurity or disorder in a dataset. A node with all examples of one class has entropy = 0 (perfectly pure). A node with equal examples of two classes has entropy = 1 (maximum impurity).

Entropy Formula (★ MUST MEMORIZE ★)
H(S) = −Σ pᵢ · log₂(pᵢ)
For binary classification: H(S) = −p₊·log₂(p₊) − p₋·log₂(p₋)
pᵢ = proportion of examples belonging to class i
log₂(0) is taken as 0 by convention. H = 0 means pure node. H = 1 means maximally impure (50-50 split).
Information Gain (IG) — Splitting Criterion for ID3
IG(S, A) = H(S) − Σᵥ (|Sᵥ|/|S|) · H(Sᵥ)
S = dataset, A = attribute/feature, v = each value of A, Sᵥ = subset where A=v
Choose the attribute with highest Information Gain to split on first.

Worked Example — Entropy Calculation

Dataset for emotion prediction:
Color: G(3S), B(2H 1S?), R(4H 1S?)

P(H|Wig=Y) = 1/2, P(S|Wig=Y) = 1/2

For H(Emotion | Masks=3): [from paper, only 1 example with masks=3: R,Y,3→H]
P(H|Masks=3) = 1/1 = 1

KEY LOG VALUES TO REMEMBER:
log₂(1/2) = -1   → -(1/2)·(-1) = 0.5 per term
log₂(1/4) = -2   → -(1/4)·(-2) = 0.5
log₂(2/3) ≈ -0.585 → -(2/3)·(-0.585) = 0.390
log₂(1/3) ≈ -1.585 → -(1/3)·(-1.585) = 0.528
H(2/3,1/3) = 0.390 + 0.528 = 0.918 bits

Key Measure 2 — Gini Impurity (CART Algorithm)

Gini Impurity is an alternative splitting measure used by the CART algorithm. Unlike entropy (uses log), Gini uses squares — faster to compute.

Gini Impurity Formula (★ MUST MEMORIZE ★)
Gini(S) = 1 − Σ pᵢ²
For binary: Gini(S) = 1 − p₊² − p₋² = 2·p·(1−p)
Pure node (all one class): Gini = 0. Equal split (50-50): Gini = 0.5 (max). Choose split with lowest weighted Gini.
Weighted Gini for a Split on Attribute A
Gini_split(S, A) = Σᵥ (|Sᵥ|/|S|) · Gini(Sᵥ)
Choose attribute A that gives minimum Gini_split (= maximum purity improvement)

Fast Answer Trick — Gini Special Cases

If a node has only ONE class → Gini = 0 (write directly, no calculation needed). If a node is 50-50 binary → Gini = 0.5 (max). If 2/3 one class, 1/3 another → Gini = 1−(4/9)−(1/9) = 4/9 ≈ 0.444. Memorize Gini(2/3,1/3) = 4/9 and Gini(3/4,1/4) = 3/8 for common splits!

Decision Tree Algorithm (ID3) — Full Steps

1
Calculate entropy/Gini of the complete dataset (root entropy).
2
For each attribute: Calculate information gain (or Gini reduction) for splitting on that attribute.
3
Select best attribute: Choose attribute with max IG (or min Gini). This becomes the root/node.
4
Split the dataset into subsets based on each value of the selected attribute.
5
Recurse: For each subset, repeat steps 1–4 until: (a) all examples in subset have same class (pure leaf), (b) no attributes left, or (c) empty subset.
6
Leaf assignment: Assign majority class to leaf node.

Overfitting & Pruning

Decision trees tend to overfit — they memorize training data perfectly but generalize poorly. Pruning reduces tree size:

  • Pre-pruning: Stop growing tree early (set max depth, min samples per leaf)
  • Post-pruning: Grow full tree, then remove branches that don't improve validation accuracy

Advantages & Disadvantages (Theory Filler)

AdvantagesDisadvantages
Easy to interpret (visual tree)Overfits easily on noisy data

Handles both numeric and categoricalBiased towards features with more values

Topic: Decision Tree Algorithm · Unit 2
Decision Tree Algorithm (ID3):
A Decision Tree is a hierarchical model that partitions the feature space using a series of binary or multi-way splits based on feature values. It is a non-parametric supervised learning method that can handle both classification and regression tasks.

Algorithm Steps:
1. Compute entropy H(S) of the complete dataset
2. For each attribute A, compute Information Gain: IG(S,A) = H(S) − Σ(|Sᵥ|/|S|)·H(Sᵥ)
3. Select attribute with maximum IG as the splitting node
4. Partition dataset into subsets based on attribute values
5. Recursively apply to each subset until stopping criteria met

Entropy:
Measures disorder/impurity of a set. H(S) = −Σpᵢ·log₂(pᵢ)
H = 0 → pure set (all same class). H = 1 → perfectly mixed (equal split).
Example: {5 Yes, 5 No} → H = −(0.5)log₂(0.5) − (0.5)log₂(0.5) = 1 bit

Information Gain:
Measures how much entropy is reduced by splitting on an attribute.
IG(S,A) = H(S) − Σᵥ (|Sᵥ|/|S|)·H(Sᵥ)
Higher IG → more useful attribute for splitting → chosen first.

Example: If H(root)=1.0 and splitting on "Outlook" gives weighted entropy=0.693, then IG(Outlook)=1.0−0.693=0.307.

K-Nearest Neighbor (KNN)

70% · Numerical

What is KNN?

K-Nearest Neighbor (KNN) is a lazy learning algorithm — it stores all training data and makes predictions at runtime by finding the k most similar (nearest) training examples and taking a vote (classification) or average (regression).

Intuition: "Tell me who your friends are and I'll tell you who you are." If 3 of your 5 nearest neighbors are doctors, you're probably a doctor too. KNN classifies a new point based on the majority class of its k closest training points.

KNN Distance Metrics — MUST KNOW
Euclidean: d(x,y) = √[Σᵢ (xᵢ − yᵢ)²] ← Most common (use for numeric)
Manhattan: d(x,y) = Σᵢ |xᵢ − yᵢ| ← City block distance
Minkowski: d(x,y) = (Σᵢ |xᵢ − yᵢ|ᵖ)^(1/p) ← General (p=2→Euclidean, p=1→Manhattan)
For Categorical features: Hamming distance = (# positions where values differ) / total positions

KNN Algorithm Steps

1
Choose k (usually odd to avoid ties). Common: k=3, k=5.
2
Calculate distance between the query point and ALL training points using chosen metric.
3
Sort distances in ascending order. Select the k points with smallest distances (nearest neighbors).
4
Classify: Assign the majority class among the k neighbors (for regression: take average of k neighbors' values).

Categorical Distance (2024 Part B Q4)

For non-numeric attributes, we can't use Euclidean distance. Use overlap metric:

d(xᵢ, yᵢ) = 0 if xᵢ = yᵢ (same value → zero distance)
d(xᵢ, yᵢ) = 1 if xᵢ ≠ yᵢ (different value → distance = 1)
Total Hamming distance = Σᵢ d(xᵢ, yᵢ)
Example: (Sunny, Hot, High, Weak) vs (Overcast, Hot, High, Strong): differ in positions 1 and 4 → distance = 2

Choosing k — The Bias-Variance Tradeoff in KNN

k valueBiasVarianceEffect
k=1LowHighOverfitting — sensitive to noise
k=largeHighLowUnderfitting — too smooth decision boundary
k=√nMediumMediumTypical rule of thumb

Q. How distance be computed for attributes that are not numeric, but are categorical in k-nearest neighbor classifier? Give example.

Topic: KNN Categorical Distance · Unit 2
For categorical (nominal) attributes, we cannot use Euclidean or Minkowski distance since there's no meaningful numerical distance between categories like "Red" and "Blue."

Method 1 — Simple Match (Overlap Metric / Hamming Distance):
d(xᵢ, yᵢ) = 0 if values match, 1 if they don't.
Total distance = count of mismatched attributes.

Example:
Point A: (Sunny, Hot, High, Weak)

Uses class distribution similarity. d(a,b) = Σc |P(class=c|A=a) − P(class=c|A=b)|²
Considers how similarly each value relates to the class label.

Method 3 — One-Hot Encoding:
Convert categorical to binary vectors, then apply Euclidean distance on the encoded vectors. Example: Outlook {Sunny,Overcast,Rainy} → [1,0,0], [0,1,0], [0,0,1]. Then use Euclidean on the binary vectors.

Logistic Regression

65% · Theory

What is Logistic Regression?

Despite the name, Logistic Regression is a classification algorithm, not a regression one. It predicts the probability that an input belongs to a class. The output is always between 0 and 1, so it's perfect for binary classification (Yes/No, Spam/Not-Spam).

Why not Linear Regression for classification? Linear regression can produce values like -0.5 or 1.7 — meaningless as probabilities. Logistic regression squashes any real number into [0,1] using the sigmoid function.

Sigmoid (Logistic) Function — Core Formula
σ(z) = 1 / (1 + e⁻ᶻ) where z = β₀ + β₁x₁ + β₂x₂ + ... + βₖxₖ
Output: σ(z) → 0 as z → −∞, σ(z) → 0.5 when z=0, σ(z) → 1 as z → +∞
Decision boundary: Predict class 1 if σ(z) ≥ 0.5 (i.e., z ≥ 0), else predict class 0.
Logistic Regression Model
P(Y=1 | X) = σ(Xβ) = 1 / (1 + e^(−(β₀ + β₁x₁ + ...)))
Log-odds (logit): ln[P(Y=1)/P(Y=0)] = β₀ + β₁x₁ + ... + βₖxₖ
Training: Maximize log-likelihood using gradient descent (no closed-form solution like linear regression).

Logistic vs Linear Regression

AspectLinear RegressionLogistic Regression
OutputAny real number ℝProbability [0,1]
TaskRegression (continuous prediction)Classification (binary/multi-class)
Functionŷ = Xβ (identity)P = σ(Xβ) (sigmoid)
Loss functionMSE (Mean Squared Error)Cross-entropy (Binary Log Loss)
Decision boundaryN/AP = 0.5 → Xβ = 0 (hyperplane)

Connection to SVM

Both Logistic Regression and SVM find a decision boundary (hyperplane) for binary classification. Logistic Regression uses maximum likelihood (probabilistic) and outputs probabilities. SVM uses maximum margin (geometric) and outputs class labels directly. LR works better when you need calibrated probabilities. SVM works better in high-dimensional spaces with small datasets. Support Vector Machine (SVM)

🔥 85% — Part A/B/C

What is SVM?

SVM (Support Vector Machine) is a powerful classification algorithm that finds the optimal hyperplane to separate classes. The key idea: instead of just any hyperplane that separates classes, SVM finds the one that maximizes the margin between classes.

Intuition: Imagine two groups of people on a field. You draw a line separating them. The SVM line is NOT just any line — it's the one drawn exactly midway between the nearest person from each group, maximizing the gap between them. Those nearest people are called support vectors.

Fig 7.1 — SVM finds the hyperplane that maximizes the margin. Double-circled points are Support Vectors.

Maximum Marginal Hyperplane (MMH) — 2024 Part A Q1

The Maximum Marginal Hyperplane (MMH) is the decision boundary that is equidistant from the nearest data points of each class, with the maximum possible distance (margin) between them.

SVM Hyperplane and Margin
Hyperplane: w·x + b = 0 (decision boundary)
Support Vectors: w·x + b = +1 (positive margin boundary)
Support Vectors: w·x + b = −1 (negative margin boundary)
Margin = 2 / ||w||
Maximizing margin = minimizing ||w||². SVM solves: min (1/2)||w||² subject to yᵢ(w·xᵢ+b) ≥ 1 for all i

Why Maximize Margin? (Theory Filler)

A larger margin means the classifier is more confident about its predictions and generalizes better to new data. Points far from the boundary can be misclassified and the model still works. Points close to the boundary are the risky ones — the support vectors.

SVM Optimization Problem (Primal Form)
Minimize: (1/2)||w||²
Subject to: yᵢ(w·xᵢ + b) ≥ 1 for all i = 1,...,n
This is a convex quadratic program — globally optimal solution guaranteed. Solved using Lagrange multipliers (dual form).

Soft Margin SVM — Slack Variables (2023 Part B Q5)

Real data is rarely linearly separable. Soft Margin SVM allows some misclassification by introducing slack variables ξᵢ ≥ 0 that measure how much a point violates the margin constraint.

Soft Margin Optimization (with Slack Variables)
Minimize: (1/2)||w||² + C · Σᵢ ξᵢ
Subject to: yᵢ(w·xᵢ + b) ≥ 1 − ξᵢ and ξᵢ ≥ 0
ξᵢ = 0: correctly classified with margin ≥ 1
0 < ξᵢ ≤ 1: inside margin but correct side
ξᵢ > 1: misclassified
C = regularization parameter. Large C → hard margin (penalize violations heavily). Small C → soft margin (allow more violations).

When SVM vs Random Forest? (2023 Part B Q5)

Use SVM whenUse Random Forest when
High-dimensional data (text, genomics)Large datasets (SVM is slow on big data)
Clear margin of separation existsNoisy data with many irrelevant features
Small to medium datasetsNeed feature importance rankings
Need a global optimal solutionNeed interpretable model

Kernel Trick — Non-Linear SVM

When data is not linearly separable, SVM uses a kernel function to map data to a higher-dimensional space where it becomes linearly separable — without explicitly computing the transformation.

Common Kernel Functions
Linear Kernel: K(x,z) = x·z
Polynomial Kernel: K(x,z) = (γ·x·z + r)ᵈ
RBF (Gaussian) Kernel: K(x,z) = exp(−γ·||x−z||²) ← Most popular
Sigmoid Kernel: K(x,z) = tanh(γ·x·z + r)
RBF kernel works for most problems. Use Linear for high-dimensional text data. Use Polynomial for image classification.

Q. Explain the purpose of slack variables in SVM problem formulation. When would you use SVM over random forest and vice-versa?

Topic: SVM Slack Variables · Unit 2
Purpose of Slack Variables in SVM:
In hard-margin SVM, we require all points to be correctly classified outside the margin. But real data is often noisy or not linearly separable, making this impossible.

Slack variables ξᵢ ≥ 0 are introduced to allow some violations:
• ξᵢ = 0: point correctly classified outside margin (no violation)
• 0 < ξᵢ < 1: point inside the margin but on correct side
• ξᵢ = 1: point exactly on the decision boundary
• ξᵢ > 1: point misclassified (violation)

Modified objective: Minimize (1/2)||w||² + C·Σξᵢ
Subject to: yᵢ(w·xᵢ+b) ≥ 1−ξᵢ, ξᵢ ≥ 0

The parameter C controls the trade-off: large C = penalize misclassification heavily (hard margin, may overfit). Small C = allow more violations (soft margin, better generalization).

Random Forest Algorithm

70% · Theory + Concepts

What is Random Forest?

Random Forest is an ensemble learning method that builds many decision trees and combines their predictions. The "random" refers to two sources of randomness: (1) each tree is trained on a random bootstrap sample of the training data, (2) each split considers only a random subset of features.

Intuition: Ask 100 experts (decision trees) independently. Each expert has seen different data and considers different factors. Take a vote. The crowd wisdom (ensemble) is much more accurate and robust than any single expert.

Random Forest — Two Key Concepts
1. Bagging (Bootstrap Aggregating): Each tree trained on random sample WITH REPLACEMENT
2. Feature Subsampling: Each split considers random m features out of M total (m ≈ √M)
Final prediction: Majority vote (classification) | Average (regression)
These two sources of randomness ensure trees are different → diverse ensemble → reduced variance → better generalization

Random Forest Algorithm — Full Steps

1
Set parameters: Choose number of trees T (e.g., 100), feature subset size m (typically √M for classification).
2
For each tree t = 1 to T:
a) Draw bootstrap sample Dₜ of size n from training data D (with replacement)
b) Grow a decision tree on Dₜ: at each node, randomly select m features; split on best feature among those m
c) Grow tree fully (no pruning needed — ensemble handles overfitting)
3
Prediction: For new point x, run through all T trees. Final prediction = majority class vote (or average for regression).

Advantages of Random Forest

  • Reduces overfitting compared to single decision tree (lower variance)
  • Works well with high-dimensional data and large datasets
  • Robust to outliers and noise
  • Provides feature importance scores (which features matter most)
  • No feature scaling required (same as decision trees)
  • Parallelizable — trees can be built simultaneously

Disadvantages

  • Less interpretable than a single decision tree (black box)
  • Computationally expensive — builds many trees
  • Large memory requirements for storing all trees
  • Slower prediction than single tree

Decision Tree Algorithm: Builds a flowchart-like tree by recursively splitting data based on the most informative feature.
