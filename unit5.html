Semi-Supervised Learning

70% В· Part A/B

What is Semi-Supervised Learning?

In the real world, labelling data is expensive and time-consuming. Semi-supervised learning (SSL) sits between supervised learning (all data labelled) and unsupervised learning (no labels). It uses a small amount of labelled data combined with a large amount of unlabelled data for training.

Intuition: Imagine you're a teacher. You have 10 answered exam papers (labelled) and 1000 blank papers (unlabelled). SSL uses the patterns from those 1000 blank papers to improve learning from just the 10.

Fig 1.1 вҖ” Semi-supervised sits between supervised and unsupervised learning

Key Assumptions (Why SSL Works)

  • Smoothness Assumption: If two points are close in the input space, their labels should be similar. Unlabelled data helps identify the "smooth" regions.
  • Cluster Assumption: Data forms natural clusters. Points in the same cluster likely share the same label. Unlabelled data reveals cluster structure.
  • Manifold Assumption: High-dimensional data lies on a low-dimensional manifold. Unlabelled data helps map this manifold accurately.

Approaches to SSL

  • Self-Training: Train on labelled data вҶ’ predict labels for unlabelled data вҶ’ add high-confidence predictions to training set вҶ’ retrain. Repeat.
  • Co-Training: Train two classifiers on different feature views of the same data. Each classifier labels data for the other.
  • Graph-Based Methods: Build a graph where nodes = data points, edges = similarity. Propagate labels from labelled nodes to unlabelled nodes.
  • Generative Models: Learn distribution P(x) from unlabelled data, then use P(x|class) for classification.

Advantages

  • Reduces labelling cost significantly (crucial for medical images, legal documents)
  • Better generalization than supervised-only on small labelled datasets
  • Exploits the large amounts of cheap unlabelled data available on the internet
рҹ”—
Connection to Unit 2 & 3

SSL connects supervised learning (Unit 2 вҖ” labelled data, Decision Trees, SVM) with unsupervised learning (Unit 3 вҖ” K-means clustering, GMM). The "cluster assumption" of SSL directly uses K-means clustering to discover structure in unlabelled data. Think of SSL as: clustering (Unit 3) + classification (Unit 2) combined smartly.

What is Reinforcement Learning?

85% В· Foundation

The Core Idea

Reinforcement Learning (RL) is a type of machine learning where an agent learns to make decisions by interacting with an environment to maximize cumulative reward. There is no teacher telling the agent what to do вҖ” it learns purely from trial and error.

Real-life analogy: A child learning to walk. No one gives the child a manual. The child tries, falls (negative reward), adjusts, tries again, and eventually learns to balance (positive reward). The child is the agent; the floor is the environment.

Markov Decision Process (MDP)

рҹ”Ҙ 95% вҖ” Most Important

What is MDP?

A Markov Decision Process is the mathematical framework that formalizes the RL problem. It provides a structured way to model decision-making in situations where outcomes are partly random and partly under the control of an agent.

The word "Markov" means the future depends only on the present state, NOT the history of past states. This is the Markov Property.

Markov Property (Most Important!)
P(sвӮңвӮҠвӮҒ | sвӮң, aвӮң, sвӮңвӮӢвӮҒ, aвӮңвӮӢвӮҒ, ...) = P(sвӮңвӮҠвӮҒ | sвӮң, aвӮң)
The next state depends ONLY on the current state and action. The past doesn't matter. This is called "memoryless" property.

MDP вҖ” Formal Definition (5-Tuple)

An MDP is defined by a 5-tuple: (S, A, P, R, Оі)

SymbolNameDefinition
SState SpaceSet of all possible states
AAction SpaceSet of all possible actions
P(s'|s,a)Transition ProbabilityProbability of reaching state s' from s by taking action a
R(s,a,s')Reward FunctionImmediate reward received after transitioning from s to s' via a
ОіDiscount FactorОі вҲҲ [0,1]. How much future rewards are worth now
Return (Total Discounted Reward)
GвӮң = rвӮңвӮҠвӮҒ + ОіВ·rвӮңвӮҠвӮӮ + ОіВІВ·rвӮңвӮҠвӮғ + ... = ОЈвӮ–вӮҢвӮҖ^вҲһ ОібөҸ В· rвӮңвӮҠвӮ–вӮҠвӮҒ
Оі=0: only care about immediate reward. Оі=1: care equally about all future rewards. Typical Оі = 0.9
рҹ§ 
Memory Trick вҖ” Discount Factor Оі

Think of Оі as inflation. вӮ№1 today вү  вӮ№1 next year. With Оі=0.9, a reward of 10 earned 2 steps later is worth 0.9ВІ Г— 10 = 8.1 today. The further in the future, the less it's worth NOW. This prevents the agent from chasing infinitely long reward sequences.

Policy and Value Function

A policy ПҖ is a rule that tells the agent what action to take in each state. It can be deterministic (ПҖ(s) = a) or stochastic (ПҖ(a|s) = probability).

State Value Function V^ПҖ(s)
V^ПҖ(s) = E_ПҖ[GвӮң | SвӮң = s] = E_ПҖ[ОЈвӮ–вӮҢвӮҖ^вҲһ ОібөҸ В· rвӮңвӮҠвӮ–вӮҠвӮҒ | SвӮң = s]
Expected total discounted reward starting from state s and following policy ПҖ
Action Value Function Q^ПҖ(s,a)
Q^ПҖ(s,a) = E_ПҖ[GвӮң | SвӮң = s, AвӮң = a]
Expected total discounted reward starting from state s, taking action a, then following policy ПҖ

An optimal policy ПҖ* is the policy that maximises the value function for ALL states:

Bellman Equations

рҹ”Ҙ 90% вҖ” Must Memorize

Core Idea: Recursion is the Key

The Bellman Equation expresses the value of a state as the immediate reward PLUS the discounted value of the next state. This recursive structure is the foundation of all RL algorithms.

Intuition: The value of being in a hotel room = (comfort tonight) + Оі Г— (value of tomorrow's destination you'll travel to from here).

Bellman Expectation Equation (For a Policy ПҖ)

Bellman Expectation вҖ” State Value
V^ПҖ(s) = ОЈвӮҗ ПҖ(a|s) ОЈ_{s'} P(s'|s,a) [R(s,a,s') + ОіВ·V^ПҖ(s')]
For deterministic policy ПҖ: V^ПҖ(s) = ОЈ_{s'} P(s'|s,ПҖ(s)) [R + ОіV^ПҖ(s')]
Bellman Expectation вҖ” Action Value
Q^ПҖ(s,a) = ОЈ_{s'} P(s'|s,a) [R(s,a,s') + Оі ОЈ_{a'} ПҖ(a'|s') Q^ПҖ(s',a')]

Relationship between V and Q

V^ПҖ(s) = ОЈвӮҗ ПҖ(a|s) В· Q^ПҖ(s,a)
Q^ПҖ(s,a) = R(s,a) + Оі ОЈ_{s'} P(s'|s,a) V^ПҖ(s')
V = weighted sum of Q values. Q = immediate reward + discounted V of next state.

вӯҗ Bellman Optimality Equation (The Golden Formula)

The optimal value function V* satisfies the Bellman Optimality Equation вҖ” it tells you the best you can possibly do from any state:

Bellman Optimality вҖ” State Value V*
V*(s) = max_a ОЈ_{s'} P(s'|s,a) [R(s,a,s') + ОіВ·V*(s')]
V* is the maximum V possible. The "max" replaces the weighted sum over policy.
Bellman Optimality вҖ” Action Value Q*
Q*(s,a) = ОЈ_{s'} P(s'|s,a) [R(s,a,s') + ОіВ·max_{a'} Q*(s',a')]
This is the Q-Learning update target! Q-Learning directly approximates Q* using this equation.
вҡЎ
Memory Trick вҖ” Bellman vs Q-Learning

Bellman Optimality Equation IS Q-Learning conceptually. In Q-Learning, the update rule is:
Q(s,a) вҶҗ Q(s,a) + Оұ[r + ОіВ·max_{a'} Q(s',a') вҲ’ Q(s,a)]
The bracketed term is TD Error = actual вҲ’ predicted. The target r + ОіВ·max Q(s',a') IS the Bellman equation. Always write this in your answer!

Markov Reward Process (MRP) вҖ” 2024 Part A Q10

An MRP is an MDP with no actions вҖ” just states, transitions, and rewards. It models stochastic environments where the agent has no control.

MRP Bellman Equation
V(s) = R(s) + Оі В· ОЈ_{s'} P(s'|s) В· V(s')
No action term since MRP has no decisions. MDP = MRP + Actions.

An MRP is a tuple (S, P, R, Оі) вҖ” a Markov chain (random walk through states) augmented with a reward signal. Policy Evaluation вҖ” Monte Carlo

85% В· Part B

What is Monte Carlo in RL?

Monte Carlo (MC) methods learn from complete episodes of experience. Instead of using Bellman equations (which require a model), MC estimates value functions by averaging the actual returns observed from real or simulated episodes.

Key idea: Play out an entire episode from start to finish. Calculate the actual total return G at the end. Update the value function using this actual return. Repeat many times вҶ’ average converges to true value.

Monte Carlo Value Update
V(s) вҶҗ V(s) + Оұ В· [G_t вҲ’ V(s)]
G_t = actual return from episode. Оұ = learning rate. After many episodes, V(s) converges to V^ПҖ(s)

Monte Carlo Policy Evaluation вҖ” Algorithm Steps

1
Initialize V(s) = 0 for all states, empty returns list Returns(s) = [] for all s.
2
Generate a complete episode following policy ПҖ: sвӮҖ, aвӮҖ, rвӮҒ, sвӮҒ, aвӮҒ, rвӮӮ, ..., sвӮҷ (terminal).
3
Calculate return G for each timestep: G_t = r_{t+1} + ОіВ·r_{t+2} + ОіВІВ·r_{t+3} + ...
4
For each state s appearing in the episode: append G_t to Returns(s). Update V(s) = mean(Returns(s)).
5
Repeat from Step 2 for many episodes until V converges.

Key Difference to Remember

DP = needs model (knows P). MC = model-free but needs complete episodes (can't do with continuing tasks). TD = model-free AND works online (updates after every step). This is why TD (Q-Learning, SARSA) is the most practical for real RL problems.

Policy Iteration & Value Iteration

рҹ”Ҙ 90% В· Part B/C

Policy Iteration

Policy Iteration finds the optimal policy by alternating between two steps: Policy Evaluation (how good is current policy?) and Policy Improvement (can we do better?).

Fig 6.1 вҖ” Policy Iteration: evaluate вҶ’ improve вҶ’ repeat until optimal
1
Initialize: Start with any policy ПҖвӮҖ (e.g., random).
2
Policy Evaluation: For current policy ПҖ, solve Bellman equation iteratively to compute V^ПҖ(s) for all s. Repeat until convergence: V^ПҖ(s) вҶҗ ОЈ_{s'} P(s'|s,ПҖ(s)) [R + ОіV^ПҖ(s')]
3
Policy Improvement: For each state, find the greedy action: ПҖ'(s) вҶҗ argmax_a ОЈ_{s'} P(s'|s,a)[R + ОіV^ПҖ(s')]
4
Check convergence: If ПҖ' = ПҖ (policy unchanged), stop вҖ” we have found ПҖ*. Otherwise, set ПҖ вҶҗ ПҖ' and go to Step 2.

Value Iteration

Value Iteration combines evaluation and improvement into a single step. Instead of fully evaluating the policy first, it applies the Bellman optimality equation directly as an update rule:

Value Iteration Update Rule
V_{k+1}(s) вҶҗ max_a ОЈ_{s'} P(s'|s,a) [R(s,a,s') + ОіВ·V_k(s')]
Repeat until |V_{k+1}(s) вҲ’ V_k(s)| < Оө for all s. Then extract ПҖ*(s) = argmax_a ОЈ_{s'} P(s'|s,a)[R + ОіV*(s')]
1
Initialize: VвӮҖ(s) = 0 for all states s.
2
Update: Apply Bellman optimality equation: V_{k+1}(s) = max_a[R(s,a) + ОіОЈ_{s'} P(s'|s,a)V_k(s')]
3
Check convergence: If max_s|V_{k+1}(s) вҲ’ V_k(s)| < Оө, stop. Otherwise increment k and repeat.
4
Extract policy: ПҖ*(s) = argmax_a[R + ОіОЈP(s'|s,a)V*(s')]

MDP: Mathematical framework for sequential decision-making. Tuple (S,A,P,R,Оі). Agent transitions between states via actions, receives rewards, seeks to maximize cumulative reward. Satisfies Markov property.

Policy Iteration:
Start with initial policy ПҖ вҶ’ Evaluate V^ПҖ fully (solve Bellman) вҶ’ Improve policy greedily (ПҖ' = argmax_a Q) вҶ’ Repeat until ПҖ stable. Guaranteed convergence to ПҖ*. Each cycle = evaluate + improve.

Merges evaluation and improvement. Apply Bellman optimality equation directly:

Repeat until V converges вҶ’ then extract optimal policy once at the end. Simpler but requires more iterations.

Q-Learning

рҹ”Ҙ 95% вҖ” Most Asked!

Core Idea вҖ” Learning Optimal Q Without a Model

Q-Learning is a model-free, off-policy TD control algorithm that directly learns the optimal action-value function Q*(s,a) without needing a model of the environment (no need to know P(s'|s,a)).

Intuition: Imagine you're exploring a maze. Every time you take a step, you update your "guess" of how good each position-direction pair is. Over time, your guesses converge to the true values. You don't need a map (model) вҖ” just your experiences.

Q-Learning Update Rule (вҳ… MUST MEMORIZE вҳ…)
Q(s,a) вҶҗ Q(s,a) + Оұ В· [r + Оі В· max_{a'} Q(s',a') вҲ’ Q(s,a)]
Оұ = learning rate (0 < Оұ вүӨ 1) | Оі = discount factor | r = reward received
TD Error = r + ОіВ·max Q(s',a') вҲ’ Q(s,a) = (target) вҲ’ (current estimate)

Q-Learning Algorithm (Full)

1
Initialize: Q(s,a) = 0 for all (s,a) pairs. Set Оұ (learning rate), Оі (discount), Оө (exploration rate).
2
For each episode: Initialize state s (start state of episode).
3
Choose action: Using Оө-greedy policy вҖ” with probability Оө choose random action; with probability (1-Оө) choose a* = argmax_a Q(s,a).
4
Take action a: Observe reward r and next state s'.
5
Update Q: Q(s,a) вҶҗ Q(s,a) + Оұ[r + ОіВ·max_{a'} Q(s',a') вҲ’ Q(s,a)]
6
Update state: s вҶҗ s'. If s' is terminal, go to Step 2. Else go to Step 3.
7
Repeat until Q converges (changes become negligible).

Оө-Greedy Exploration

Q-Learning uses Оө-greedy to balance exploration (try new things) vs exploitation (use current best knowledge):

ПҖ(s) = random action, with probability Оө (exploration)
ПҖ(s) = argmax_a Q(s,a), with probability 1вҲ’Оө (exploitation)
Оө typically starts high (0.9) and decays over time as agent gains experience.

SARSA (State-Action-Reward-State-Action)

рҹ”Ҙ 90% В· Part B/C

SARSA вҖ” The Name Tells the Algorithm

The name SARSA describes exactly what it uses in the update: State, Action, Reward, next State, next Action вҖ” a quintuple (s, a, r, s', a').

SARSA Update Rule (вҳ… MUST MEMORIZE вҳ…)
Q(s,a) вҶҗ Q(s,a) + Оұ В· [r + Оі В· Q(s',a') вҲ’ Q(s,a)]
Key difference from Q-Learning: uses Q(s',a') where a' is the action ACTUALLY CHOSEN next (not max)
a' is selected using the SAME Оө-greedy policy used for a вҶ’ on-policy

SARSA Algorithm вҖ” Full Steps

1
Initialize Q(s,a) = 0 for all s, a. Set Оұ, Оі, Оө.
2
For each episode: Initialize s. Choose a from s using Оө-greedy policy.
3
Take action a. Observe reward r and next state s'.
4
Choose a' from s' using Оө-greedy (same policy). вҶҗ This is what makes it ON-policy.
5
Update: Q(s,a) вҶҗ Q(s,a) + Оұ[r + ОіQ(s',a') вҲ’ Q(s,a)]
6
s вҶҗ s', a вҶҗ a'. If s is terminal, end episode. Else go to Step 3.

Role of SARSA in Machine Learning:

SARSA (State-Action-Reward-State-Action) is a model-free, on-policy temporal difference (TD) control algorithm used to find the optimal policy in a reinforcement learning setting.

Role/Applications:
вҖў Robot control: trains robots to navigate safely while avoiding risky actions
вҖў Game playing: learns strategies for board games, video games
вҖў Traffic control: optimizes signal timings while considering actual driver behavior
вҖў Finance: portfolio optimization under realistic trading constraints

Why SARSA specifically: SARSA is on-policy вҖ” it evaluates and improves the policy being followed during learning. This makes it more conservative and suitable for real-world deployments where taking risky exploratory actions has real consequences.

SARSA Update:
Q(sвӮң,aвӮң) вҶҗ Q(sвӮң,aвӮң) + Оұ[rвӮңвӮҠвӮҒ + ОіQ(sвӮңвӮҠвӮҒ,aвӮңвӮҠвӮҒ) вҲ’ Q(sвӮң,aвӮң)]

SARSA vs Q-Learning вҖ” Detailed Comparison:

1. Policy Type: SARSA = on-policy (behavior = target policy). Q-Learning = off-policy (behavior policy generates data; target policy = greedy)

2. Update difference:
  SARSA: uses Q(s',a') where a' ~ ПҖ(В·|s') (actually chosen action)
  Q-Learning: uses max_a' Q(s',a') (best possible action, not necessarily chosen)

3. Risk sensitivity: SARSA accounts for exploration during updates вҶ’ conservative. Q-Learning ignores exploration in update вҶ’ aggressive/optimal

4. Convergence: Both converge under standard conditions (decaying Оө, decaying Оұ, sufficient exploration). Q-Learning вҶ’ optimal Q*. SARSA вҶ’ optimal Q for the Оө-greedy policy.

5. Practical example:
Cliff Walking (grid world with cliff edge): Q-Learning learns to walk right along the cliff (optimal path). SARSA learns a safer path away from the cliff because it accounts for random exploration steps that might fall off. In simulation, Q-Learning is better. In deployment, SARSA is safer.

Methods/Variants of SARSA:

1. Standard SARSA (one-step SARSA):
Updates Q after each (s,a,r,s',a') transition. Basic on-policy TD(0) control.
Q(s,a) вҶҗ Q(s,a) + Оұ[r + ОіQ(s',a') вҲ’ Q(s,a)]

2. SARSA(О») вҖ” Eligibility Traces:
Extends SARSA to multi-step returns. Maintains eligibility trace e(s,a) for each state-action pair.
Оҙ = r + ОіQ(s',a') вҲ’ Q(s,a) (TD error)
e(s,a) вҶҗ ОіО»e(s,a) + 1 (update trace for current s,a)
Q(s,a) вҶҗ Q(s,a) + ОұВ·ОҙВ·e(s,a) for ALL (s,a)
О»=0: equivalent to standard SARSA. О»=1: equivalent to Monte Carlo.

3. n-Step SARSA:
Updates using returns from n steps ahead instead of just 1.
G_t^(n) = r_{t+1} + Оіr_{t+2} + ... + Оі^(n-1)r_{t+n} + ОівҒҝQ(s_{t+n}, a_{t+n})
Q(s,a) вҶҗ Q(s,a) + Оұ[G_t^(n) вҲ’ Q(s,a)]
n=1: SARSA. n=вҲһ: Monte Carlo.

4. Expected SARSA:
Instead of Q(s',a'), uses expected value over all possible a':
Q(s,a) вҶҗ Q(s,a) + Оұ[r + Оі ОЈ_{a'} ПҖ(a'|s')Q(s',a') вҲ’ Q(s,a)]
Reduces variance compared to standard SARSA.

Model-Based Reinforcement Learning

What is Model-Based RL?

In model-based RL, the agent builds an internal model of the environment (learns the transition function P(s'|s,a) and reward function R(s,a)). It then uses this model to plan ahead without taking actual actions.

Analogy: A chess grandmaster mentally simulates future moves before playing. The model = their knowledge of chess rules. Planning = mental simulation. In model-free RL, the agent learns only from actual experience вҖ” like a beginner who must physically try every move.

Dyna-Q вҖ” Key Model-Based Algorithm

Dyna-Q integrates model learning with Q-Learning. It interleaves real experience with simulated experience from the learned model.

1
Real experience: Take action a in real env вҶ’ observe r, s' вҶ’ update Q(s,a) (like Q-Learning).
2
Model learning: Update model: M(s,a) вҶҗ (r, s') вҖ” store what happened.
3
Planning (simulated): Repeat n times: randomly pick (s,a) from history вҶ’ query model M(s,a) вҶ’ get simulated (r,s') вҶ’ update Q(s,a) again using simulated data.

The key benefit: n simulated Q-updates per real step вҶ’ much faster convergence. AlphaGo uses model-based planning (MCTS вҖ” Monte Carlo Tree Search).
