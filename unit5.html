Semi-Supervised Learning

70% · Part A/B

What is Semi-Supervised Learning?

In the real world, labelling data is expensive and time-consuming. Semi-supervised learning (SSL) sits between supervised learning (all data labelled) and unsupervised learning (no labels). It uses a small amount of labelled data combined with a large amount of unlabelled data for training.

Intuition: Imagine you're a teacher. You have 10 answered exam papers (labelled) and 1000 blank papers (unlabelled). SSL uses the patterns from those 1000 blank papers to improve learning from just the 10.

Fig 1.1 — Semi-supervised sits between supervised and unsupervised learning

Key Assumptions (Why SSL Works)

  • Smoothness Assumption: If two points are close in the input space, their labels should be similar. Unlabelled data helps identify the "smooth" regions.
  • Cluster Assumption: Data forms natural clusters. Points in the same cluster likely share the same label. Unlabelled data reveals cluster structure.
  • Manifold Assumption: High-dimensional data lies on a low-dimensional manifold. Unlabelled data helps map this manifold accurately.

Approaches to SSL

  • Self-Training: Train on labelled data → predict labels for unlabelled data → add high-confidence predictions to training set → retrain. Repeat.
  • Co-Training: Train two classifiers on different feature views of the same data. Each classifier labels data for the other.
  • Graph-Based Methods: Build a graph where nodes = data points, edges = similarity. Propagate labels from labelled nodes to unlabelled nodes.
  • Generative Models: Learn distribution P(x) from unlabelled data, then use P(x|class) for classification.

Advantages

  • Reduces labelling cost significantly (crucial for medical images, legal documents)
  • Better generalization than supervised-only on small labelled datasets
  • Exploits the large amounts of cheap unlabelled data available on the internet
🔗
Connection to Unit 2 & 3

SSL connects supervised learning (Unit 2 — labelled data, Decision Trees, SVM) with unsupervised learning (Unit 3 — K-means clustering, GMM). The "cluster assumption" of SSL directly uses K-means clustering to discover structure in unlabelled data. Think of SSL as: clustering (Unit 3) + classification (Unit 2) combined smartly.

What is Reinforcement Learning?

85% · Foundation

The Core Idea

Reinforcement Learning (RL) is a type of machine learning where an agent learns to make decisions by interacting with an environment to maximize cumulative reward. There is no teacher telling the agent what to do — it learns purely from trial and error.

Real-life analogy: A child learning to walk. No one gives the child a manual. The child tries, falls (negative reward), adjusts, tries again, and eventually learns to balance (positive reward). The child is the agent; the floor is the environment.

Yes, RL is an appropriate abstract model for evolution.

• In evolution, organisms "act" (survive/reproduce or not) — this parallels the agent taking actions.

• Evolution operates on populations; RL on a single agent.

• Time scales are very different (evolution = millions of years; RL = episodes).

Markov Decision Process (MDP)

🔥 95% — Most Important

What is MDP?

A Markov Decision Process is the mathematical framework that formalizes the RL problem. It provides a structured way to model decision-making in situations where outcomes are partly random and partly under the control of an agent.

The word "Markov" means the future depends only on the present state, NOT the history of past states. This is the Markov Property.

Markov Property (Most Important!)
P(sₜ₊₁ | sₜ, aₜ, sₜ₋₁, aₜ₋₁, ...) = P(sₜ₊₁ | sₜ, aₜ)
The next state depends ONLY on the current state and action. The past doesn't matter. This is called "memoryless" property.

MDP — Formal Definition (5-Tuple)

An MDP is defined by a 5-tuple: (S, A, P, R, γ)

SymbolNameDefinition
SState SpaceSet of all possible states
AAction SpaceSet of all possible actions
P(s'|s,a)Transition ProbabilityProbability of reaching state s' from s by taking action a
R(s,a,s')Reward FunctionImmediate reward received after transitioning from s to s' via a
γDiscount Factorγ ∈ [0,1]. How much future rewards are worth now
Return (Total Discounted Reward)
Gₜ = rₜ₊₁ + γ·rₜ₊₂ + γ²·rₜ₊₃ + ... = Σₖ₌₀^∞ γᵏ · rₜ₊ₖ₊₁
γ=0: only care about immediate reward. γ=1: care equally about all future rewards. Typical γ = 0.9
🧠
Memory Trick — Discount Factor γ

Think of γ as inflation. ₹1 today ≠ ₹1 next year. With γ=0.9, a reward of 10 earned 2 steps later is worth 0.9² × 10 = 8.1 today. The further in the future, the less it's worth NOW. This prevents the agent from chasing infinitely long reward sequences.

Policy and Value Function

A policy π is a rule that tells the agent what action to take in each state. It can be deterministic (π(s) = a) or stochastic (π(a|s) = probability).

State Value Function V^π(s)
V^π(s) = E_π[Gₜ | Sₜ = s] = E_π[Σₖ₌₀^∞ γᵏ · rₜ₊ₖ₊₁ | Sₜ = s]
Expected total discounted reward starting from state s and following policy π
Action Value Function Q^π(s,a)
Q^π(s,a) = E_π[Gₜ | Sₜ = s, Aₜ = a]
Expected total discounted reward starting from state s, taking action a, then following policy π

RTU Example — Grid World MDP

A classic MDP example: A robot on a 3×3 grid. Goal: reach top-right corner (reward +10). Falling in pit = reward -5. Each move = -1 reward.

📌 Grid World MDP Setup
┌─────┬─────┬─────┐
│     │     │ +10 │   ← Goal state (terminal)
├─────┼─────┼─────┤
│     │ PIT │     │   ← Pit = -5 (terminal)
├─────┼─────┼─────┤
│ S   │     │     │   ← S = Start state
└─────┴─────┴─────┘

States: S = {(1,1),(1,2),...,(3,3)} = 9 states
Actions: A = {Up, Down, Left, Right}
Rewards: R = +10 at goal, -5 at pit, -1 otherwise
γ = 0.9

The agent must find the optimal policy π* that navigates from S to Goal while avoiding the Pit.

Optimal Policy — What it Means

An optimal policy π* is the policy that maximises the value function for ALL states:

π*(s) = argmax_a Q*(s,a) for all s ∈ S
π* is found when the value function converges (no more change). Guaranteed to exist for finite MDPs.

Bellman Equations

🔥 90% — Must Memorize

Core Idea: Recursion is the Key

The Bellman Equation expresses the value of a state as the immediate reward PLUS the discounted value of the next state. This recursive structure is the foundation of all RL algorithms.

Intuition: The value of being in a hotel room = (comfort tonight) + γ × (value of tomorrow's destination you'll travel to from here).

Bellman Expectation Equation (For a Policy π)

Bellman Expectation — State Value
V^π(s) = Σₐ π(a|s) Σ_{s'} P(s'|s,a) [R(s,a,s') + γ·V^π(s')]
For deterministic policy π: V^π(s) = Σ_{s'} P(s'|s,π(s)) [R + γV^π(s')]
Bellman Expectation — Action Value
Q^π(s,a) = Σ_{s'} P(s'|s,a) [R(s,a,s') + γ Σ_{a'} π(a'|s') Q^π(s',a')]

Relationship between V and Q

V^π(s) = Σₐ π(a|s) · Q^π(s,a)
Q^π(s,a) = R(s,a) + γ Σ_{s'} P(s'|s,a) V^π(s')
V = weighted sum of Q values. Q = immediate reward + discounted V of next state.

⭐ Bellman Optimality Equation (The Golden Formula)

The optimal value function V* satisfies the Bellman Optimality Equation — it tells you the best you can possibly do from any state:

Bellman Optimality — State Value V*
V*(s) = max_a Σ_{s'} P(s'|s,a) [R(s,a,s') + γ·V*(s')]
V* is the maximum V possible. The "max" replaces the weighted sum over policy.
Bellman Optimality — Action Value Q*
Q*(s,a) = Σ_{s'} P(s'|s,a) [R(s,a,s') + γ·max_{a'} Q*(s',a')]
This is the Q-Learning update target! Q-Learning directly approximates Q* using this equation.
⚡
Memory Trick — Bellman vs Q-Learning

Bellman Optimality Equation IS Q-Learning conceptually. In Q-Learning, the update rule is:
Q(s,a) ← Q(s,a) + α[r + γ·max_{a'} Q(s',a') − Q(s,a)]
The bracketed term is TD Error = actual − predicted. The target r + γ·max Q(s',a') IS the Bellman equation. Always write this in your answer!

Markov Reward Process (MRP) — 2024 Part A Q10

An MRP is an MDP with no actions — just states, transitions, and rewards. It models stochastic environments where the agent has no control.

MRP Bellman Equation
V(s) = R(s) + γ · Σ_{s'} P(s'|s) · V(s')
No action term since MRP has no decisions. MDP = MRP + Actions.

Q. Give your intuition about: (a) Markov Reward Process (MRP) (b) Bellman Equation
2024 · Part A Q10 · 2 marks
Topic: MRP, Bellman Equation · Unit 5
(a) Markov Reward Process (MRP):
An MRP is a tuple (S, P, R, γ) — a Markov chain (random walk through states) augmented with a reward signal. The agent receives rewards R(s) in each state and transitions to next states with probability P(s'|s). There are no actions — the process is fully stochastic. V*(s) = max_a Σ_{s'} P(s'|s,a)[R + γV*(s')]
Intuition: "The value of where I am = what I get now + discounted value of where I'll end up." It breaks the complex future into manageable recursive steps.

Policy Evaluation — Monte Carlo

85% · Part B

What is Monte Carlo in RL?

Monte Carlo (MC) methods learn from complete episodes of experience. Instead of using Bellman equations (which require a model), MC estimates value functions by averaging the actual returns observed from real or simulated episodes.

Key idea: Play out an entire episode from start to finish. Calculate the actual total return G at the end. Update the value function using this actual return. Repeat many times → average converges to true value.

Monte Carlo Value Update
V(s) ← V(s) + α · [G_t − V(s)]
G_t = actual return from episode. α = learning rate. After many episodes, V(s) converges to V^π(s)

Monte Carlo Policy Evaluation — Algorithm Steps

1
Initialize V(s) = 0 for all states, empty returns list Returns(s) = [] for all s.
2
Generate a complete episode following policy π: s₀, a₀, r₁, s₁, a₁, r₂, ..., sₙ (terminal).
3
Calculate return G for each timestep: G_t = r_{t+1} + γ·r_{t+2} + γ²·r_{t+3} + ...
4
For each state s appearing in the episode: append G_t to Returns(s). Update V(s) = mean(Returns(s)).
5
Repeat from Step 2 for many episodes until V converges.

Key Difference to Remember

DP = needs model (knows P). MC = model-free but needs complete episodes (can't do with continuing tasks). TD = model-free AND works online (updates after every step). This is why TD (Q-Learning, SARSA) is the most practical for real RL problems.

Policy Evaluation using Monte Carlo — Steps:

Monte Carlo policy evaluation estimates V^π(s) — the value of each state under policy π — by averaging observed returns from complete episodes.

Algorithm:
Input: Policy π to evaluate
Initialize: V(s) = 0 ∀s; Returns(s) = {} ∀s

Policy Iteration & Value Iteration

🔥 90% · Part B/C

Policy Iteration

Policy Iteration finds the optimal policy by alternating between two steps: Policy Evaluation (how good is current policy?) and Policy Improvement (can we do better?).

Fig 6.1 — Policy Iteration: evaluate → improve → repeat until optimal
1
Initialize: Start with any policy π₀ (e.g., random).
2
Policy Evaluation: For current policy π, solve Bellman equation iteratively to compute V^π(s) for all s. Repeat until convergence: V^π(s) ← Σ_{s'} P(s'|s,π(s)) [R + γV^π(s')]
3
Policy Improvement: For each state, find the greedy action: π'(s) ← argmax_a Σ_{s'} P(s'|s,a)[R + γV^π(s')]
4
Check convergence: If π' = π (policy unchanged), stop — we have found π*. Otherwise, set π ← π' and go to Step 2.

Value Iteration

Value Iteration combines evaluation and improvement into a single step. Instead of fully evaluating the policy first, it applies the Bellman optimality equation directly as an update rule:

Value Iteration Update Rule
V_{k+1}(s) ← max_a Σ_{s'} P(s'|s,a) [R(s,a,s') + γ·V_k(s')]
Repeat until |V_{k+1}(s) − V_k(s)| < ε for all s. Then extract π*(s) = argmax_a Σ_{s'} P(s'|s,a)[R + γV*(s')]
1
Initialize: V₀(s) = 0 for all states s.
2
Update: Apply Bellman optimality equation: V_{k+1}(s) = max_a[R(s,a) + γΣ_{s'} P(s'|s,a)V_k(s')]
3
Check convergence: If max_s|V_{k+1}(s) − V_k(s)| < ε, stop. Otherwise increment k and repeat.
4
Extract policy: π*(s) = argmax_a[R + γΣP(s'|s,a)V*(s')]

Q-Learning

🔥 95% — Most Asked!

Core Idea — Learning Optimal Q Without a Model

Q-Learning is a model-free, off-policy TD control algorithm that directly learns the optimal action-value function Q*(s,a) without needing a model of the environment (no need to know P(s'|s,a)).

Intuition: Imagine you're exploring a maze. Every time you take a step, you update your "guess" of how good each position-direction pair is. Over time, your guesses converge to the true values. You don't need a map (model) — just your experiences.

Q-Learning Update Rule (★ MUST MEMORIZE ★)
Q(s,a) ← Q(s,a) + α · [r + γ · max_{a'} Q(s',a') − Q(s,a)]
α = learning rate (0 < α ≤ 1) | γ = discount factor | r = reward received
TD Error = r + γ·max Q(s',a') − Q(s,a) = (target) − (current estimate)

Q-Learning Algorithm (Full)

1
Initialize: Q(s,a) = 0 for all (s,a) pairs. Set α (learning rate), γ (discount), ε (exploration rate).
2
For each episode: Initialize state s (start state of episode).
3
Choose action: Using ε-greedy policy — with probability ε choose random action; with probability (1-ε) choose a* = argmax_a Q(s,a).
4
Take action a: Observe reward r and next state s'.
5
Update Q: Q(s,a) ← Q(s,a) + α[r + γ·max_{a'} Q(s',a') − Q(s,a)]
6
Update state: s ← s'. If s' is terminal, go to Step 2. Else go to Step 3.
7
Repeat until Q converges (changes become negligible).

ε-Greedy Exploration

Q-Learning uses ε-greedy to balance exploration (try new things) vs exploitation (use current best knowledge):

π(s) = random action, with probability ε (exploration)
π(s) = argmax_a Q(s,a), with probability 1−ε (exploitation)
ε typically starts high (0.9) and decays over time as agent gains experience.

⭐ RTU Numerical Example — Q-Learning

📌 Q-Learning Walkthrough

Simple 3-state MDP: States = {A, B, Goal}. Actions = {stay, move}.

Initial Q-table:
       stay   move
A    [  0,    0  ]
B    [  0,    0  ]

Given: α=0.5, γ=0.9, Episode: A→move→B (r=0), B→move→Goal (r=10)

Step 1: State=A, Action=move, r=0, Next=B
  Q(A,move) ← 0 + 0.5[0 + 0.9·max(Q(B)) − 0]
            ← 0 + 0.5[0 + 0.9·0] = 0

Step 2: State=B, Action=move, r=10, Next=Goal (terminal)
  max Q(Goal) = 0 (terminal state)
  Q(B,move) ← 0 + 0.5[10 + 0.9·0 − 0]
            ← 0.5 × 10 = 5

Updated Q-table after episode 1:
       stay   move
A    [  0,    0  ]
B    [  0,    5  ]

Episode 2 — Step 1: State=A, Action=move, r=0, Next=B
  Q(A,move) ← 0 + 0.5[0 + 0.9·max(0,5) − 0]
            ← 0 + 0.5[0 + 4.5] = 2.25

After more episodes, Q(A,move) converges toward 0.9×10 = 9, Q(B,move)→10.
⚡
Exam Validation Trick

After Q-Learning converges, verify: Q*(s,a) = R + γ·max Q*(s'). For terminal state: Q(Goal,*) = 0. Work backwards to check your table. Also verify: optimal policy = always choose action with max Q in each state. If your final policy reaches the goal, your Q-table is likely correct.

SARSA (State-Action-Reward-State-Action)

🔥 90% · Part B/C

SARSA — The Name Tells the Algorithm

The name SARSA describes exactly what it uses in the update: State, Action, Reward, next State, next Action — a quintuple (s, a, r, s', a').

SARSA Update Rule (★ MUST MEMORIZE ★)
Q(s,a) ← Q(s,a) + α · [r + γ · Q(s',a') − Q(s,a)]
Key difference from Q-Learning: uses Q(s',a') where a' is the action ACTUALLY CHOSEN next (not max)
a' is selected using the SAME ε-greedy policy used for a → on-policy

SARSA Algorithm — Full Steps

1
Initialize Q(s,a) = 0 for all s, a. Set α, γ, ε.
2
For each episode: Initialize s. Choose a from s using ε-greedy policy.
3
Take action a. Observe reward r and next state s'.
4
Choose a' from s' using ε-greedy (same policy). ← This is what makes it ON-policy.
5
Update: Q(s,a) ← Q(s,a) + α[r + γQ(s',a') − Q(s,a)]
6
s ← s', a ← a'. If s is terminal, end episode. Else go to Step 3.

Q. What is the role of State-Action-Reward-State-Action (SARSA) in machine learning? Differentiate between SARSA and Q-learning.
May 2025 · Part C Q2 · 10 marks
Topic: SARSA, Q-Learning comparison · Unit 5
Role of SARSA in Machine Learning:

SARSA (State-Action-Reward-State-Action) is a model-free, on-policy temporal difference (TD) control algorithm used to find the optimal policy in a reinforcement learning setting.

Role/Applications:
• Robot control: trains robots to navigate safely while avoiding risky actions
• Game playing: learns strategies for board games, video games
• Traffic control: optimizes signal timings while considering actual driver behavior
• Finance: portfolio optimization under realistic trading constraints

Why SARSA specifically: SARSA is on-policy — it evaluates and improves the policy being followed during learning. This makes it more conservative and suitable for real-world deployments where taking risky exploratory actions has real consequences.

SARSA Update:
Q(sₜ,aₜ) ← Q(sₜ,aₜ) + α[rₜ₊₁ + γQ(sₜ₊₁,aₜ₊₁) − Q(sₜ,aₜ)]
where aₜ₊₁ is the action ACTUALLY chosen in sₜ₊₁.

SARSA vs Q-Learning — Detailed Comparison:

1. Policy Type: SARSA = on-policy (behavior = target policy). Q-Learning = off-policy (behavior policy generates data; target policy = greedy)

2. Update difference:
  SARSA: uses Q(s',a') where a' ~ π(·|s') (actually chosen action)
  Q-Learning: uses max_a' Q(s',a') (best possible action, not necessarily chosen)

3. Risk sensitivity: SARSA accounts for exploration during updates → conservative. Q-Learning ignores exploration in update → aggressive/optimal

4. Convergence: Both converge under standard conditions (decaying ε, decaying α, sufficient exploration). Q-Learning → optimal Q*. SARSA → optimal Q for the ε-greedy policy.

5. Practical example:
Cliff Walking (grid world with cliff edge): Q-Learning learns to walk right along the cliff (optimal path). SARSA learns a safer path away from the cliff because it accounts for random exploration steps that might fall off. In simulation, Q-Learning is better. In deployment, SARSA is safer.
Q. Explain various methods of SARSA.
June 2022 · Part B Q6b · 4 marks
Topic: SARSA Methods · Unit 5
Methods/Variants of SARSA:

1. Standard SARSA (one-step SARSA):
Updates Q after each (s,a,r,s',a') transition. Basic on-policy TD(0) control.
Q(s,a) ← Q(s,a) + α[r + γQ(s',a') − Q(s,a)]

2. SARSA(λ) — Eligibility Traces:
Extends SARSA to multi-step returns. Maintains eligibility trace e(s,a) for each state-action pair.
δ = r + γQ(s',a') − Q(s,a) (TD error)
e(s,a) ← γλe(s,a) + 1 (update trace for current s,a)
Q(s,a) ← Q(s,a) + α·δ·e(s,a) for ALL (s,a)
λ=0: equivalent to standard SARSA. λ=1: equivalent to Monte Carlo.

3. n-Step SARSA:
Updates using returns from n steps ahead instead of just 1.
G_t^(n) = r_{t+1} + γr_{t+2} + ... + γ^(n-1)r_{t+n} + γⁿQ(s_{t+n}, a_{t+n})
Q(s,a) ← Q(s,a) + α[G_t^(n) − Q(s,a)]
n=1: SARSA. n=∞: Monte Carlo.

4. Expected SARSA:
Instead of Q(s',a'), uses expected value over all possible a':
Q(s,a) ← Q(s,a) + α[r + γ Σ_{a'} π(a'|s')Q(s',a') − Q(s,a)]
Reduces variance compared to standard SARSA.

Model-Based Reinforcement Learning

75% · Part B

What is Model-Based RL?

In model-based RL, the agent builds an internal model of the environment (learns the transition function P(s'|s,a) and reward function R(s,a)). It then uses this model to plan ahead without taking actual actions.

Analogy: A chess grandmaster mentally simulates future moves before playing. The model = their knowledge of chess rules. Planning = mental simulation. In model-free RL, the agent learns only from actual experience — like a beginner who must physically try every move.

AspectModel-Free RLModel-Based RL
Needs model?No (learns from experience only)Yes (learns/uses P and R)
Sample efficiencyLow (needs many samples)High (can simulate)
Computational costLow (just interact)High (planning + learning)
Model errorNot affectedModel errors → suboptimal policy
ExamplesQ-Learning, SARSADyna-Q, World Models, AlphaZero

Dyna-Q — Key Model-Based Algorithm

Dyna-Q integrates model learning with Q-Learning. It interleaves real experience with simulated experience from the learned model.

1
Real experience: Take action a in real env → observe r, s' → update Q(s,a) (like Q-Learning).
2
Model learning: Update model: M(s,a) ← (r, s') — store what happened.
3
Planning (simulated): Repeat n times: randomly pick (s,a) from history → query model M(s,a) → get simulated (r,s') → update Q(s,a) again using simulated data.

The key benefit: n simulated Q-updates per real step → much faster convergence. AlphaGo uses model-based planning (MCTS — Monte Carlo Tree Search).

🔗
Connection — RL Methods Summary

The whole RL landscape: DP (has model, no learning needed — Policy/Value Iteration) → Model-Based RL (learns model, then plans) → Model-Free RL (no model — Monte Carlo for complete episodes; TD/Q-Learning/SARSA for step-by-step). RTU exam pattern: always compare at least two of these in every Part B/C question about RL.

Q. What do you mean by reinforcement learning? Also explain the model based reinforcement learning.
May 2025 · Part B Q2 · 4 marks
Topic: RL, Model-Based RL · Unit 5
Reinforcement Learning:
RL is a type of ML where an agent learns to take actions in an environment to maximize cumulative reward. The agent is not told what actions to take; it must discover the best actions by trial and error. The feedback is a scalar reward signal — positive for good actions, negative for bad. Key components: Agent, Environment, State, Action, Reward, Policy, Value function.

Model-Based Reinforcement Learning:
In model-based RL, the agent builds an internal model of the environment — specifically:
• Transition model: P(s'|s,a) — what happens when action a is taken in state s
• Reward model: R(s,a) — what reward is received

The agent uses this learned model to plan — simulate future transitions without taking real actions — and then uses planning to update its policy/value function.

Advantages:
• Highly sample-efficient (fewer real interactions needed)
• Can plan ahead like a chess player thinking ahead

Disadvantages:
• Model errors can propagate and lead to suboptimal policy ("model bias")
• Building accurate models of complex environments is hard

Key Algorithm — Dyna-Q:
Combines Q-Learning with model learning. For each real experience, performs n planning steps using the learned model → dramatically speeds up learning.
